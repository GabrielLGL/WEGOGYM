# Perf audit — Groupes B (lazy loading) et C (memoization)

## Statut : ✅ Résolu — 20260307 (React.lazy sur 20 screens + useMemo/useCallback déployés)

## Contexte
Le perf audit `docs/bmad/prompts/20260305-2300-perf-audit-A.md` a defini 3 groupes :
- **Groupe A** : FlatList optimization — DONE
- **Groupe B** : Lazy loading des screens secondaires
- **Groupe C** : Memoization des composants et callbacks couteux

## Actions — Groupe B (Lazy loading)
1. Identifier les screens secondaires (Settings, Charts, etc.)
2. Implementer `React.lazy()` + `Suspense` avec fallback `<ScreenLoader />`
3. Verifier que la navigation reste fluide

## Actions — Groupe C (Memoization)
1. Identifier les re-renders inutiles (React DevTools Profiler)
2. Ajouter `React.memo()` sur les composants de liste (ExerciseCard, SessionCard, etc.)
3. Ajouter `useMemo()` / `useCallback()` sur les calculs couteux et callbacks
4. Verifier avec Profiler que les re-renders sont reduits

## Metriques attendues
- Reduction du TTI (Time to Interactive) sur HomeScreen
- Moins de re-renders sur les listes longues
- Navigation plus fluide entre les screens

## Risques
- `React.lazy` peut ne pas bien fonctionner avec React Navigation — tester
- Over-memoization peut rendre le code plus complexe sans gain reel
- Tester sur un device mid-range pour mesurer l'impact reel

## Priorite : IMPORTANT
Ameliore l'experience utilisateur, surtout sur devices bas de gamme.
