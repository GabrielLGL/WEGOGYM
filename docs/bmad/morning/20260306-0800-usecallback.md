# Rapport — useCallback manquants — 2026-03-06

## Problème
`handleValidateSet` et `handleConfirmEnd` dans WorkoutScreen ne sont pas wrappés dans `useCallback`, ce qui cause des re-renders inutiles des composants enfants (FlatList items).

## Fichiers concernés
- `mobile/src/screens/WorkoutScreen.tsx`

## Commande à lancer
/do docs/bmad/morning/20260306-0800-usecallback.md

## Contexte
Le commit b708518 a ajouté useCallback sur 3 handlers (handleClose, handleConfirmAbandon, handleValidateSet) mais handleValidateSet et handleConfirmEnd restent sans useCallback selon le dernier rapport verrif. Vérifier l'état actuel et wrapper les handlers manquants.

## Critères de validation
- Tous les handlers passés en props à des composants enfants sont wrappés dans useCallback
- `npx tsc --noEmit` : 0 erreurs
- `npm test` : tous les tests passent

## Statut
✅ Résolu — 20260306-2210

## Résolution
Les deux handlers (`handleConfirmEnd` L209, `handleValidateSet` L376) sont déjà wrappés dans `useCallback`. Tous les handlers passés en props sont memoizés (`handleClose`, `handleConfirmAbandon`, `handleValidateSet`, `handleConfirmEnd`, `renderWorkoutItem`).
