# Rapport — _raw unsafe access — 2026-03-06

## Problème
2 fichiers accèdent à `_raw` directement sur des modèles WatermelonDB, ce qui contourne le typage TypeScript et peut casser si le schéma change.

## Fichiers concernés
- `mobile/src/screens/WorkoutScreen.tsx`
- `mobile/src/screens/HistoryDetailScreen.tsx`

## Commande à lancer
/do docs/bmad/morning/20260306-0800-raw-unsafe.md

## Contexte
`_raw` est un accès interne WatermelonDB qui retourne les données brutes SQLite. Il faut remplacer par les accesseurs typés du modèle (ex: `exercise.name` au lieu de `exercise._raw.name`). Si `unsafeFetchRaw()` est utilisé pour la perf, c'est acceptable mais doit être typé avec une interface explicite.

## Critères de validation
- Aucun accès `._raw.` non typé dans les 2 fichiers
- `npx tsc --noEmit` : 0 erreurs
- `npm test` : tous les tests passent
