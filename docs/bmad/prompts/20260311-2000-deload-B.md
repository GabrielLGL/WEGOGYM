## Contexte technique

### Pattern i18n
Les fichiers `fr.ts` et `en.ts` sont des objets plats imbriqués. **Ne PAS utiliser `as const`** (cf MEMORY.md).
Accès via `useLanguage().t` → ex: `t.deload.title`.

Pattern d'interpolation existant (utilisé dans motivational phrases) :
```typescript
// fr.ts
motivational: {
  returning: 'Content de te revoir ! Ça fait {days} jours.',
}
// Utilisation : t.motivational.returning.replace('{days}', String(days))
```

### Pattern composant card (HomeScreen)
Le HomeScreen utilise des cards avec ce style pattern :
```tsx
<View style={[styles.card, { backgroundColor: colors.card }]}>
  <Text style={[styles.cardTitle, { color: colors.text }]}>{title}</Text>
  <Text style={[styles.cardSubtitle, { color: colors.textSecondary }]}>{subtitle}</Text>
</View>
```

Utiliser :
- `useColors()` hook pour les couleurs dynamiques (dark/light)
- `useLanguage()` hook pour les traductions
- `spacing.*`, `borderRadius.*`, `fontSize.*` depuis `theme/index.ts`
- Pas de couleurs hardcodées

### Types de la recommandation (créés par Groupe A)
```typescript
type DeloadType = 'rest_day' | 'deload_week' | 'reduce_volume' | 'muscle_overload'
type DeloadSeverity = 'warning' | 'suggestion'

interface DeloadRecommendation {
  type: DeloadType
  severity: DeloadSeverity
  reasonKey: string
  reasonParams?: Record<string, string | number>
  affectedMuscles?: string[]
}
```

**IMPORTANT** : Ce groupe peut créer ces types localement dans le composant si le Groupe A
n'a pas encore tourné. Les types seront reconciliés lors de l'intégration (Groupe C).

### 3. Créer `DeloadRecommendationCard.tsx`

```tsx
// Props
interface DeloadRecommendationCardProps {
  recommendation: DeloadRecommendation  // ou le type local
  onDismiss: () => void
}
```

**Design du composant :**
- Card avec `backgroundColor: colors.card` + bordure gauche colorée :
  - `warning` → `colors.danger` (#FF3B30) bordure
  - `suggestion` → `colors.primary` bordure
- Icône en haut à gauche (emoji ou Ionicons) :
  - `rest_day` → 😴 ou `bed-outline`
  - `deload_week` → 📉 ou `trending-down-outline`
  - `reduce_volume` → ⚠️ ou `warning-outline`
  - `muscle_overload` → 💪 ou `fitness-outline`
- Titre (gras, `fontSize.md`) : `t.deload.[type]Title`
- Message (normal, `fontSize.sm`, `colors.textSecondary`) : `t.deload.[reasonKey]` avec interpolation
- Si `affectedMuscles` → afficher les muscles en chips (style texte, pas ChipSelector)
- Bouton "Compris" (`ghost` variant ou simple TouchableOpacity) aligné à droite
- Animation d'entrée : `FadeIn` simple avec `Animated.View` opacity (pas Reanimated)

**Règles de style :**
- `borderRadius.md` (14)
- `spacing.md` (16) padding
- `spacing.sm` (8) gap entre éléments
- Utiliser `useColors()` pour TOUTES les couleurs
- Utiliser `useLanguage()` pour TOUTES les chaînes

**Pas de `<Modal>` natif** — c'est un composant inline dans le scroll, pas un overlay.
