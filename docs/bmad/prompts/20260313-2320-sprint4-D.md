## Implémentation

### prPredictionHelpers.ts

```ts
interface PRDataPoint {
  dateMs: number    // set.createdAt.getTime()
  weight: number    // poids soulevé
  reps: number
  orm: number       // poids * (1 + reps / 30) — Epley 1RM
}

interface PRPrediction {
  currentBest1RM: number          // meilleur 1RM actuel
  predicted1RM: number            // 1RM prédit dans weeksAhead semaines
  targetWeight: number            // prochain palier rond (arrondi à 2.5 kg)
  weeksToTarget: number           // semaines estimées pour atteindre targetWeight
  weeklyGainRate: number          // gain moyen en 1RM par semaine (kg/semaine)
  dataPoints: number              // nombre de PRs utilisés pour la régression
  confidence: 'low' | 'medium' | 'high'  // 'low' si < 3 PRs, 'medium' si 3-5, 'high' si > 5
}

/**
 * Régression linéaire simple sur les 1RM historiques des PRs.
 * Retourne null si moins de 2 PRs disponibles.
 */
export function computePRPrediction(sets: WorkoutSet[]): PRPrediction | null
```

**Algorithme :**
1. Filtrer les sets avec `isPr === true`
2. Si moins de 2 PRs → retourner `null`
3. Calculer 1RM (Epley) pour chaque PR : `orm = weight * (1 + reps / 30)`
4. Extraire `{ x: dateMs, y: orm }` pour chaque PR (utiliser `createdAt.getTime()`)
5. Régression linéaire sur les paires `(x, y)` :
   - `slope = Σ((x-x̄)(y-ȳ)) / Σ((x-x̄)²)` en kg/ms
   - `intercept = ȳ - slope * x̄`
   - Convertir slope en kg/semaine : `slope * 7 * 24 * 60 * 60 * 1000`
6. Calculer `weeklyGainRate` (kg/semaine)
7. Si `weeklyGainRate <= 0` → utiliser la moyenne des 3 derniers PRs comme proxy (0.5 kg/semaine fallback)
8. `currentBest1RM` = max(orm) de tous les PRs
9. `targetWeight` = prochain multiple de 2.5 kg au-dessus de `currentBest1RM` : `Math.ceil(currentBest1RM / 2.5) * 2.5 + 2.5`... en fait : prochain PR = 1RM actuel + 2.5kg (charge réelle) ou ceil à +5%
   **Simplification :** `targetWeight = Math.ceil((currentBest1RM * 1.025) / 2.5) * 2.5`
10. `weeksToTarget = Math.round((targetWeight - currentBest1RM) / Math.max(0.1, weeklyGainRate))`
11. `confidence` : 'low' si < 3, 'medium' si 3-5, 'high' si > 5

## Contraintes
- Pas de migration DB — uniquement `setsForExercise` déjà dans les props
- Pas de dépendances npm supplémentaires
