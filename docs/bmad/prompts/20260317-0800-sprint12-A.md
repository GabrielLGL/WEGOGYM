## Implementation

### repMaxHelpers.ts

```ts
export interface RepMaxEstimate {
  epley1RM: number
  brzycki1RM: number
  average1RM: number    // moyenne des deux
  bestSet: { weight: number; reps: number }
  estimated3RM: number
  estimated5RM: number
}

export interface RepMaxHistory {
  date: Date
  estimated1RM: number
}

/**
 * Calcule le 1RM estime a partir d'un set (poids x reps).
 * Formule Epley : 1RM = weight * (1 + reps / 30)
 * Formule Brzycki : 1RM = weight * (36 / (37 - reps))
 * Si reps === 1, retourne le poids directement.
 * Si reps > 12, les formules deviennent imprecises — marquer comme "estime".
 */
export function computeRepMax(weight: number, reps: number): { epley: number; brzycki: number }

/**
 * Trouve le meilleur 1RM estime parmi une liste de sets.
 * Ignore les sets avec reps > 15 (trop imprecis).
 * Retourne null si aucun set valide.
 */
export function getBestRepMax(
  sets: Array<{ weight: number; reps: number; createdAt: Date | number }>
): RepMaxEstimate | null

/**
 * Calcule l'historique du 1RM par semaine (meilleur 1RM estime de la semaine).
 * Retourne les 12 dernieres semaines avec donnees.
 */
export function getRepMaxHistory(
  sets: Array<{ weight: number; reps: number; createdAt: Date | number }>
): RepMaxHistory[]

/**
 * Calcule les estimations de sous-maxima.
 * 3RM = 1RM * 0.93
 * 5RM = 1RM * 0.87
 * 8RM = 1RM * 0.80
 * 10RM = 1RM * 0.75
 */
export function getSubMaxEstimates(oneRepMax: number): Record<string, number>
```

**Algorithme getBestRepMax :**
1. Filtrer sets avec reps >= 1 && reps <= 15 && weight > 0
2. Pour chaque set, calculer epley et brzycki
3. Prendre la moyenne (epley + brzycki) / 2
4. Retourner le set avec la plus haute moyenne
5. Calculer aussi 3RM et 5RM a partir du meilleur 1RM
