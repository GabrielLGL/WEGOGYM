# Rapport — Collection d'Exercices (#82) — Groupe C — 20260314-0400

## Objectif
Créer un écran `ExerciseCollectionScreen` qui affiche la collection personnelle d'exercices de l'utilisateur : "X/N exercices découverts". Chaque exercice pratiqué au moins une fois est "débloqué", les autres sont verrouillés. Accessible depuis StatsScreen. Gamifié à la façon d'un Pokédex.

## Implémentation

### Logique (dans useMemo)

```ts
interface CollectionEntry {
  exercise: Exercise
  discovered: boolean     // true si au moins 1 set pour cet exercice
  setsCount: number       // nombre total de sets
  lastDoneMs: number | null
}
```

**Calcul :**
1. Construire `Set<string>` des exercice IDs utilisés (depuis sets)
2. Pour chaque exercice, construire un `CollectionEntry`
3. Trier : exercices découverts en premier (par dernière utilisation), puis non-découverts (par nom alphabétique)
4. `discoveredCount` = nombre d'exercices avec `discovered === true`

**Détails UI :**
- Exercice découvert : `colors.text` (nom), `colors.textSecondary` (muscles + count), fond `colors.card`
- Exercice non découvert : `colors.textMuted` (nom masqué "?????????"), fond `colors.cardSecondary`, icône `🔒`
- Pour les non-découverts, masquer le nom avec "?" répété selon la longueur : `exercise.name.replace(/./g, '?')`
- Nombre de colonnes : `numColumns={2}` dans FlatList
- Barre de progression en haut : `View` de largeur proportionnelle, couleur `colors.primary`

### withObservables pattern
```ts
const enhance = withObservables([], () => ({
  sets: database.get<WorkoutSet>('sets').query(

      Q.where('deleted_at', null),
      Q.or(Q.where('is_abandoned', null), Q.where('is_abandoned', false)),
    ))
  ).observe(),
  exercises: database.get<Exercise>('exercises').query().observe(),
}))
```
