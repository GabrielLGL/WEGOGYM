# Rapport — Audit 5 Site Kore — Groupe C : Focus Visible Styles — 20260227-1900

## Objectif
Ajouter des styles `focus-visible` globaux dans `globals.css`.
Le mot-clé `focus` est **complètement absent** de globals.css → les utilisateurs naviguant au clavier (Tab) ne voient aucun indicateur de focus sur les éléments interactifs (liens, boutons, inputs).

## Fichiers concernés
- `web/src/app/globals.css`

## Contexte technique
- CSS global chargé par le layout Next.js
- Le site utilise des CSS custom properties (variables) : `var(--accent)`, `var(--bg)`, etc.
- Tailwind CSS v4 inclus mais aucun utilitaire `focus-ring` utilisé sur les composants
- `prefers-reduced-motion: reduce` est déjà géré dans le fichier ✅
- Ne pas modifier les règles existantes

## Étapes

### 1. Ajouter un reset `outline: none` sur `:focus` (non-clavier)

Le navigateur affiche un outline bleu par défaut sur `:focus` (y compris click). On le supprime pour le remplacer par `:focus-visible` uniquement (qui ne se déclenche que pour la navigation clavier).

```css
/* Supprimer outline par défaut sur focus souris/touch */
*:focus:not(:focus-visible) {
  outline: none;
}
```

### 2. Ajouter le style `focus-visible` global

```css
/* Focus visible — navigation clavier uniquement */
*:focus-visible {
  outline: 2px solid var(--accent);
  outline-offset: 3px;
  border-radius: 4px;
}
```

### 3. Cas spéciaux : inputs et boutons neumorphiques

Les inputs et boutons ont un style `shadow-neu-in` / `shadow-neu-out`. L'outline doit s'adapter :

```css
/* Focus visible sur inputs (style neumorphique) */
input:focus-visible,
textarea:focus-visible {
  outline: 2px solid var(--accent);
  outline-offset: 2px;
  border-radius: 999px; /* correspond au rounded-full des inputs */
}

/* Focus visible sur boutons ronds */
button:focus-visible,
a:focus-visible {
  outline: 2px solid var(--accent);
  outline-offset: 3px;
  border-radius: 8px;
}
```

### 4. Emplacement dans globals.css

Ajouter ces règles dans la section `:root` ou juste après la section des animations (avant `@media prefers-reduced-motion`), dans la partie des utilitaires globaux.

## Contraintes
- Ne pas modifier les règles existantes
- Ne pas ajouter `outline: none` en global (bloquerait la navigation clavier)
- Utiliser uniquement `:focus-visible` (pas `:focus`) pour éviter l'outline sur les clics souris
- `var(--accent)` est défini pour les thèmes light ET dark — le focus sera toujours visible

## Critères de validation
- Navigation Tab sur la page → outline violet visible sur chaque élément interactif (liens, boutons, inputs)
- Click souris → aucun outline affiché (non-clavier, non-intrusif)
- Lighthouse Accessibility → "Interactive elements have visible focus indicators"
- Pas de régression visuelle sur le design neumorphique
