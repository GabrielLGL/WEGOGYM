# Rapport — Deload / Périodisation — Groupe A (Core Logic) — 20260311-2000

## Objectif
Créer le moteur de détection de fatigue et de recommandation de deload/repos.
Ce module est purement logique (pas d'UI). Il analyse les données existantes (histories, sets, user)
et retourne une recommandation typée.

## Étapes

### 1. Ajouter les constantes de deload dans `constants.ts`

```typescript
// --- Deload / Periodization ---
export const DELOAD_CONSECUTIVE_DAYS_THRESHOLD = 6    // >6 jours consécutifs → suggérer repos
export const DELOAD_VOLUME_SPIKE_RATIO = 1.3           // volume >130% de la moyenne 4 semaines → alerte
export const DELOAD_VOLUME_WEEKS_WINDOW = 4             // fenêtre de comparaison en semaines
export const DELOAD_MIN_HISTORY_WEEKS = 3               // minimum de semaines d'historique pour recommander
export const DELOAD_TRAINING_BLOCK_WEEKS = 6            // après 6+ semaines sans pause → suggérer deload
export const DELOAD_REST_GAP_DAYS = 2                   // moins de 2 jours de repos en 7 jours → alerte

// MRV (Maximum Recoverable Volume) — sets/semaine par muscle
// Source : Renaissance Periodization guidelines, adapté par niveau
export const MRV_THRESHOLDS: Record<string, Record<string, number>> = {
  beginner:     { chest: 14, back: 16, shoulders: 14, biceps: 12, triceps: 12, legs: 16, abs: 16 },
  intermediate: { chest: 18, back: 20, shoulders: 18, biceps: 16, triceps: 16, legs: 20, abs: 20 },
  advanced:     { chest: 22, back: 24, shoulders: 22, biceps: 20, triceps: 20, legs: 24, abs: 24 },
}
```

### 2. Créer `deloadHelpers.ts`

Interface de sortie :
```typescript
export type DeloadType = 'rest_day' | 'deload_week' | 'reduce_volume' | 'muscle_overload'
export type DeloadSeverity = 'warning' | 'suggestion'

export interface DeloadRecommendation {
  type: DeloadType
  severity: DeloadSeverity
  reasonKey: string           // clé i18n (ex: 'deload.consecutiveDays')
  reasonParams?: Record<string, string | number>  // paramètres pour interpolation
  affectedMuscles?: string[]  // muscles surchargés (pour muscle_overload)
}
```

Fonction principale :
```typescript
export function computeDeloadRecommendation(params: {
  histories: Array<{ startTime: number; endTime?: number; isAbandoned?: boolean; deletedAt?: number }>
  weeklyVolumes: number[]       // volume par semaine (dernières N semaines, index 0 = plus récent)
  setsPerMuscle?: Record<string, number>  // sets cette semaine par muscle
  userLevel: string             // 'beginner' | 'intermediate' | 'advanced'
  currentStreak: number         // semaines consécutives
  now?: number                  // pour les tests
}): DeloadRecommendation | null
```

**Signaux à détecter (par priorité)** :

1. **Jours consécutifs d'entraînement** (>= DELOAD_CONSECUTIVE_DAYS_THRESHOLD)
   - Filtrer histories non supprimées, non abandonnées
   - Grouper par jour calendaire (`Math.floor(startTime / DAY_MS)`)
   - Compter les jours consécutifs jusqu'à aujourd'hui
   - Si >= seuil → `{ type: 'rest_day', severity: 'warning', reasonKey: 'deload.consecutiveDays' }`

2. **Spike de volume** (volume semaine courante > 130% moyenne 4 semaines précédentes)
   - Utiliser `weeklyVolumes[0]` vs moyenne de `weeklyVolumes[1..4]`
   - Si ratio > DELOAD_VOLUME_SPIKE_RATIO → `{ type: 'reduce_volume', severity: 'warning' }`

3. **Block d'entraînement long** (>= DELOAD_TRAINING_BLOCK_WEEKS semaines consécutives)
   - Calculer le nombre de semaines consécutives avec au moins 1 session
   - Si >= seuil → `{ type: 'deload_week', severity: 'suggestion' }`

4. **Surcharge musculaire** (sets > MRV pour un muscle)
   - Comparer `setsPerMuscle` vs `MRV_THRESHOLDS[userLevel]`
   - Si un muscle dépasse → `{ type: 'muscle_overload', severity: 'suggestion', affectedMuscles: [...] }`

5. **Manque de repos** (< 2 jours off dans les 7 derniers jours)
   - Compter les jours avec workout dans les 7 derniers jours
   - Si > 5 jours sur 7 → `{ type: 'rest_day', severity: 'suggestion' }`

**Retourner la recommandation de plus haute priorité (1 seule).**
Si aucun signal → retourner `null`.
