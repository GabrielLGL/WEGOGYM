# Rapport ‚Äî Analyse Temps de Repos (#109) ‚Äî Groupe D ‚Äî 20260317-0800

## Objectif
Creer un nouvel ecran `StatsRestTimeScreen` analysant les temps de repos entre les series. Les timestamps des sets permettent de calculer la duree reelle entre deux series consecutives dans une meme seance. Affiche la moyenne par exercice, la tendance, et des recommandations.

- `mobile/src/model/utils/restTimeAnalysisHelpers.ts` ‚Äî NOUVEAU : calcul des temps de repos
- `mobile/src/screens/StatsRestTimeScreen.tsx` ‚Äî NOUVEAU : ecran
- `mobile/src/navigation/index.tsx` ‚Äî MODIFIER : ajouter route

## Contexte technique

### Stack & Architecture
- React Native + Expo 52 + TypeScript strict (Fabric / New Architecture)
- `useColors()` / `useLanguage()` hooks
- Les `sets` ont un champ `created_at` (timestamp de creation) ‚Äî c'est le moment ou le set est enregistre
- Temps de repos = difference entre `created_at` du set N et `created_at` du set N-1 pour le meme `history_id`
- Filtrer les repos aberrants (< 10s ou > 600s = 10min)

## Implementation

### restTimeAnalysisHelpers.ts

```ts
export interface RestTimeEntry {
  exerciseName: string
  exerciseId: string
  averageRest: number      // secondes
  minRest: number
  maxRest: number
  sampleCount: number      // nombre de repos mesures
  recommendation: 'short' | 'optimal' | 'long'
}

export interface RestTimeResult {
  entries: RestTimeEntry[]
  globalAverage: number    // moyenne globale en secondes
  totalSamples: number
}

/**
 * Calcule les temps de repos moyens par exercice.
 *
 * @param sets ‚Äî tous les sets avec timestamps, groupes par history_id
 * @param exercises ‚Äî map exerciceId -> nom
 *
 * Algorithme :
 * 1. Grouper les sets par history_id
 * 2. Dans chaque groupe, trier par created_at
 * 3. Pour chaque paire consecutive (set N, set N+1), calculer la difference en secondes
 * 4. Filtrer : garder seulement 10s <= repos <= 600s
 * 5. Grouper par exerciseId et calculer moyenne/min/max
 * 6. Recommandation basee sur la moyenne :
 *    - < 60s ‚Üí 'short'
 *    - 60-180s ‚Üí 'optimal'
 *    - > 180s ‚Üí 'long'
 * 7. Trier par nombre d'echantillons decroissant
 */
export function computeRestTimeAnalysis(
  sets: Array<{
    exerciseId: string
    historyId: string
    createdAt: Date | number
  }>,
  exercises: Array<{ id: string; name: string }>,
): RestTimeResult

/**
 * Formate les secondes en "Xm Xs" ou "Xs".
 */
export function formatRestTime(seconds: number): string
```

function getBadgeColor(rec: 'short' | 'optimal' | 'long', colors: any): string {
  switch (rec) {
