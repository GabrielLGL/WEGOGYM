## Features choisies
Toutes proviennent du backlog `docs/FEATURES.md` — section "❌ Non implémentées".
Critères de sélection :
- **Zéro migration DB** : calculé depuis les données existantes (sets, histories, user)
- **Indépendantes** : aucune dépendance croisée
- **Visibles** : apportent une valeur utilisateur immédiate
- **Scope maîtrisé** : 1-3 fichiers principaux par feature

## Instructions pour chaque Claude

### Ce que chaque Claude DOIT faire :
1. Lire SON rapport en entier avant de commencer
2. Lire les fichiers mentionnés dans "Fichiers concernés" (pour comprendre les patterns existants)
3. Écrire le code selon les spécifications exactes du rapport
4. Lancer `npx tsc --noEmit` et corriger toute erreur TypeScript
5. Faire `git add` UNIQUEMENT sur ses propres fichiers + son rapport
6. Vérifier avec `git diff --cached --name-only` avant de commit
7. Si `git push` échoue → `git pull --rebase` puis re-push
8. Sauvegarder un rapport dans `docs/bmad/do/[YYYYMMDD-HHmm]-feat-[nom].md`

### Ce que chaque Claude NE DOIT PAS faire :
- `git add .` ou `git add -A` — INTERDIT
- Modifier des fichiers non listés dans son rapport
- Créer des migrations DB (toutes les features sont DB-free)
- Ajouter des dépendances npm (toutes les libs nécessaires sont déjà installées)
