# Rapport — Tendances Composition Corporelle (#102) — Groupe B — 20260315-1630

## Objectif
Créer un écran `StatsBodyCompScreen` affichant les tendances des mesures corporelles (poids, tour de taille, hanches, poitrine, bras) sous forme de mini-graphiques sparkline sur 30/90/180 jours. Chaque mesure montre la valeur actuelle, le delta, et une courbe de tendance. Accessible depuis StatsScreen.

## Implémentation

### bodyCompTrendsHelpers.ts

```ts
export type BodyCompMetric = 'weight' | 'waist' | 'hips' | 'chest' | 'arms'

export interface BodyCompTrend {
  metric: BodyCompMetric
  label: string                  // clé i18n
  unit: string                   // 'kg' ou 'cm'
  currentValue: number | null
  previousValue: number | null   // valeur au début de la période
  delta: number | null           // current - previous
  deltaPercent: number | null
  trend: 'up' | 'down' | 'stable'
  dataPoints: Array<{ date: number; value: number }>  // pour sparkline
  hasData: boolean
}

export type BodyCompPeriod = 30 | 90 | 180

/**
 * Calcule les tendances des mesures corporelles sur une période donnée.
 */
export function computeBodyCompTrends(
  measurements: Array<{
    weight: number; waist: number; hips: number;
    chest: number; arms: number; date: number;
  }>,
  periodDays: BodyCompPeriod,
): BodyCompTrend[]
```

**Algorithme :**
1. Filtrer les mesures dans la période
2. Pour chaque métrique (weight, waist, hips, chest, arms) :
   - Extraire les dataPoints (date + value) — exclure les valeurs 0
   - `currentValue` = dernière mesure
   - `previousValue` = première mesure de la période
   - Calculer delta et deltaPercent
   - `trend` : delta > 1% → 'up', delta < -1% → 'down', sinon 'stable'
3. `hasData` = au moins 2 dataPoints
4. Trier : métriques avec données en premier
