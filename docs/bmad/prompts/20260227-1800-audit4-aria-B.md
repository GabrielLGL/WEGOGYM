# Rapport — Audit 4 Site Kore — Groupe B : Accessibilité PricingSection & PrivacyPage — 20260227-1800

## Objectif
Corriger 3 problèmes d'accessibilité dans PricingSection.tsx et privacy/page.tsx :
1. Checkmarks `&#10003;` sans `aria-hidden` → screenreaders lisent "crochet de vérification" pour chaque feature
2. Cartes pricing en `<div>` → `<article>` (landmark sémantique)
3. Flèches "←" des back-links sans `aria-hidden` → screenreaders lisent "flèche gauche"

## Fichiers concernés
- `web/src/components/sections/PricingSection.tsx`
- `web/src/app/privacy/page.tsx`

## Contexte technique
- `PricingSection` : Server Component pur — PAS de `"use client"`
- `privacy/page.tsx` : Server Component avec `export const metadata`
- `npx tsc --noEmit` (depuis `web/`) doit rester à 0 erreur

---

## Problème 1 — PricingSection : checkmarks sans aria-hidden

**Actuel :**
```tsx
<li key={feature} className="flex items-start gap-3 text-sm">
  <span className="text-[var(--accent)] mt-0.5">&#10003;</span>
  <span>{feature}</span>
</li>
```

**Problème :** `&#10003;` (✓) est lu "crochet de vérification" par les screenreaders pour chaque feature dans la liste. Le texte de la feature suffit — le checkmark est purement décoratif.

**Fix :**
```tsx
<span className="text-[var(--accent)] mt-0.5" aria-hidden="true">&#10003;</span>
```

---

## Problème 2 — PricingSection : cartes en `<div>` → `<article>`

**Actuel :**
```tsx
<div
  key={plan.name}
  className={`reveal bg-[var(--bg)] rounded-[30px] p-8 ...`}
>
```

**Problème :** Un `<div>` n'a pas de sémantique. Les cartes de pricing sont des contenus indépendants et autosuffisants — elles méritent le rôle `<article>` (landmark HTML5). Les screenreaders peuvent naviguer entre les articles avec des raccourcis clavier.

**Fix :** Remplacer `<div key={...} className={...}>` par `<article key={...} className={...}>` et fermer avec `</article>` (à la place de `</div>`).

Le reste du contenu interne (highlighted badge, h3, prix, features, CTA) : **inchangé**.

---

## Problème 3 — privacy/page.tsx : flèches "←" sans aria-hidden

**Actuel (x2 — top et bottom du fichier) :**
```tsx
<Link href="/" ...>
  <span className="group-hover:-translate-x-1 transition-transform inline-block">
    ←
  </span>
  Retour à l&apos;accueil
</Link>
```

**Problème :** Le `<span>` contenant "←" est lu par les screenreaders comme "flèche gauche, Retour à l'accueil". Le texte "Retour à l'accueil" suffit — la flèche est décorative.

**Fix :** Ajouter `aria-hidden="true"` sur les deux spans fléchés :
```tsx
<span className="group-hover:-translate-x-1 transition-transform inline-block" aria-hidden="true">
  ←
</span>
```
Les deux occurrences (haut et bas de page) doivent être corrigées.
