# Rapport — Partage workout/PR/badge — Groupe B — 20260311-2130

## Objectif
Créer le composant visuel `ShareCard` — une carte stylée qui sert de base pour la capture d'image partageable. Trois variantes : workout summary, badge, et PR.

## Fichiers concernés
- `mobile/src/components/ShareCard.tsx` (NOUVEAU)

## Contexte technique
- **Theme** : utiliser `useColors()` de `contexts/ThemeContext.tsx` pour les couleurs dynamiques (dark/light).
- **Couleurs** : `colors.background`, `colors.card`, `colors.primary`, `colors.text`, `colors.textSecondary`, `colors.accent`, `colors.danger`.
- **Spacing** : `spacing.xs` (4), `spacing.sm` (8), `spacing.ms` (12), `spacing.md` (16), `spacing.lg` (24), `spacing.xl` (32).
- **BorderRadius** : `borderRadius.sm` (10), `borderRadius.md` (14), `borderRadius.lg` (20).
- **FontSize** : `fontSize.caption` (11), `fontSize.xs` (12), `fontSize.sm` (14), `fontSize.md` (16), `fontSize.lg` (18), `fontSize.xl` (20), `fontSize.xxl` (24).
- **i18n** : utiliser `useLanguage()` → `{ t }` pour tous les textes.
- **Pas de `<Modal>` natif** — ce composant n'est pas un modal, c'est juste un `View` qui sera wrappé dans un `ViewShot` par le parent.
- **Design** : neumorphisme épuré, cohérent avec le reste de l'app.
- Le composant sera wrappé par `ViewShot` dans les écrans d'intégration (Groupe C), donc il doit être un `View` standard.

### 5. Styles
- Fond : `colors.card` avec bordure subtile `colors.border`
- Coins arrondis : `borderRadius.lg` (20)
- Padding : `spacing.lg` (24)
- Taille fixe pour capture cohérente : **width 340, height auto** (adaptatif au contenu)
- Header : fond `colors.primary` léger, texte "KORE" en bold
- Footer : texte `colors.textSecondary`, fontSize.caption
- Stats : gros chiffres (`fontSize.xxl`), labels en `fontSize.xs`
- Ombres neumorphiques légères (si dark mode = ombre interne subtile)

### 6. Formatage
- Durée : convertir secondes → `Xh XXmin` ou `XXmin`
- Volume : séparateur de milliers (`8 500 kg`)
- Utiliser `t(...)` pour tous les labels

## Contraintes
- Ne PAS utiliser `<Modal>` natif
- Ne PAS utiliser de bibliothèque d'icônes externe — utiliser des emojis pour les icônes (cohérent avec badges/milestones existants qui utilisent des emojis)
- Composant fonctionnel uniquement
- TypeScript strict — pas de `any`
- `useColors()` obligatoire pour le theming
- `useLanguage()` obligatoire pour l'i18n
- Le composant doit être un `View` simple, pas de ScrollView
- Les dimensions doivent être fixes/prévisibles pour la capture ViewShot

## Critères de validation
- `npx tsc --noEmit` → zéro erreur
- `npm test` → zéro fail
- Les 3 variantes s'affichent correctement
- Pas de couleurs hardcodées
- Tous les textes passent par `t(...)`
