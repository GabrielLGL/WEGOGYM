# Rapport — Site Kore audit 2 — Groupe A — 20260227-1500
## Fichiers manquants SEO : robots.ts, sitemap.ts, not-found.tsx, privacy/page.tsx

## Objectif
Créer les 4 fichiers essentiels absents du site Next.js qui causent des problèmes SEO et UX :
1. `robots.ts` → guide le crawler Google
2. `sitemap.ts` → liste toutes les URLs indexables
3. `not-found.tsx` → page 404 brandée (actuellement page Next.js générique)
4. `privacy/page.tsx` → page politique de confidentialité (liée depuis footer et SubscribeSection mais inexistante = 404)

## Fichiers concernés
- `web/src/app/robots.ts` — à créer
- `web/src/app/sitemap.ts` — à créer
- `web/src/app/not-found.tsx` — à créer
- `web/src/app/privacy/page.tsx` — à créer

## Contexte technique
- Next.js App Router (Next 15). Pour robots/sitemap, utiliser les fichiers spéciaux Next.js (`MetadataRoute`).
- Le site est une landing page one-page à `kore-app.com`. La home `/` est la seule URL de contenu + `/privacy`.
- Design : neumorphique, CSS vars (`--bg`, `--text-main`, `--accent`, etc. dans `globals.css`)
- Thème : dark/light toggle via `data-theme` sur `<html>`.
- Couleurs respectées via CSS vars — **jamais de valeurs hardcodées**.
- Composants réutilisables disponibles : `KoreLogo` dans `@/components/KoreLogo`.
- La page layout (`layout.tsx`) inclut `metadataBase: new URL("https://kore-app.com")`.
- L'app est en français (fr-FR).
- `web/src/app/globals.css` contient les classes neumorphiques : `.shadow-neu-out`, `.shadow-neu-in`, `.gradient-text`, `.btn-liquid`.

## Étapes

### 1. `web/src/app/robots.ts`
```ts
import type { MetadataRoute } from "next";

export default function robots(): MetadataRoute.Robots {
  return {
    rules: { userAgent: "*", allow: "/" },
    sitemap: "https://kore-app.com/sitemap.xml",
  };
}
```

### 2. `web/src/app/sitemap.ts`
```ts
import type { MetadataRoute } from "next";

export default function sitemap(): MetadataRoute.Sitemap {
  return [
    {
      url: "https://kore-app.com",
      lastModified: new Date(),
      changeFrequency: "monthly",
      priority: 1,
    },
    {
      url: "https://kore-app.com/privacy",
      lastModified: new Date(),
      changeFrequency: "yearly",
      priority: 0.3,
    },
  ];
}
```

### 3. `web/src/app/not-found.tsx`
Page 404 brandée Kore avec :
- Design cohérent avec le reste du site (CSS vars, neumorphique)
- Import `KoreLogo` depuis `@/components/KoreLogo`
- Message en français : "Page introuvable" / "404"
- Lien retour vers `/` avec style `.btn-liquid`
- Pas de `"use client"` — server component pur
- Pas d'import de `ThemeToggle` ni de `BackgroundBlobs` (page simple)
- Utiliser `<main>` avec `min-h-screen flex flex-col items-center justify-center`

### 4. `web/src/app/privacy/page.tsx`
Page politique de confidentialité :
- Server component (pas de `"use client"`)
- Export `metadata` avec title "Politique de confidentialité — Kore"
- Contenu minimal légalement acceptable pour un formulaire d'inscription beta :
  - Responsable : Kore
  - Données collectées : email, prénom (optionnel)
  - Finalité : informer du lancement de l'application
  - Durée de conservation : jusqu'au lancement ou désabonnement
  - Droit de suppression : contact@kore-app.com
  - Pas de partage avec des tiers (sauf envoi email via Resend)
- Design cohérent : `max-w-[800px] mx-auto py-20 px-6`
- Lien retour vers `/` en haut
- Utiliser les CSS vars pour les couleurs

## Contraintes
- Ne pas modifier les fichiers existants
- Pas de `any` TypeScript
- Pas de couleurs hardcodées — utiliser `var(--bg)`, `var(--text-main)`, `var(--accent)` etc.
- App language = français
- Pas de dépendances supplémentaires

- Accéder à `http://localhost:3000/404` → page Kore brandée
- Accéder à `http://localhost:3000/privacy` → page politique de confidentialité
