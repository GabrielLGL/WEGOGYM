# Rapport — ui-polish — Groupe A (Couleurs & Contraste) — 20260226-2200

## Objectif
Améliorer le contraste visuel de l'app en mode dark ET light :
1. Différencier background vs card (actuellement identiques dans les deux modes → les cartes ne "flottent" pas)
2. Corriger la couleur primaire en mode light (`#00cec9` est trop clair, contraste insuffisant)
3. Ajouter des tokens pour les inputs (champs de saisie doivent se distinguer des cartes)
4. Renforcer les borders pour mieux séparer les sections

## Fichier concerné
`mobile/src/theme/index.ts`

## Contexte technique
- `useColors()` hook dans `mobile/src/contexts/ThemeContext.tsx` expose la palette active
- Tous les composants utilisent `colors.*` depuis ce hook — changer les valeurs de token change l'app entière
- Le type `ThemeColors = typeof colors` doit rester aligné avec `lightColors`
- `getThemeColors(mode)` retourne `lightColors` si mode === 'light', sinon `colors`
- **CRITIQUE** : `lightColors` doit avoir exactement les mêmes clés que `colors`

## Étapes

### 1. Dark mode — différencier background et card
Actuellement `background: '#21242b'` et `card: '#21242b'` sont identiques.
Changer :
```ts
// colors (dark)
background: '#181b21',      // plus sombre (était #21242b)
card: '#21242b',            // reste (devient le "elevated")
cardSecondary: '#282d38',   // légèrement plus clair pour contraste
border: '#363d4d',          // plus visible (était #2c3039)
separator: '#363d4d',       // idem
```

### 2. Light mode — différencier background et card + corriger primary
Actuellement `background: '#e8ecef'` et `card: '#e8ecef'` sont identiques.
Changer :
```ts
// lightColors
background: '#d8dde6',         // plus foncé pour que les cartes ressortent
card: '#eef1f5',               // blanc cassé — les cartes "flottent" sur le fond
cardSecondary: '#e4e9f0',      // légèrement plus sombre que card
primary: '#006d6b',            // dark teal — remplace #00cec9 (trop clair, ratio 1.7:1 → 6.8:1)
success: '#006d6b',            // cohérence avec primary
border: '#b8c1cc',             // plus visible
separator: '#b8c1cc',          // idem
primaryBg: 'rgba(0, 109, 107, 0.12)',  // adapté au nouveau primary light
successBg: 'rgba(0, 109, 107, 0.12)',  // idem
```

### 3. Ajouter token `input` dans les deux palettes
Ajouter une clé `input` (fond des TextInput pour les distinguer des cartes) :
```ts
// dark
input: '#16181e',   // plus sombre que card pour enfoncer visuellement le champ

// light
input: '#ffffff',   // blanc pur — contraste maximal sur fond card
```

### 4. Renforcer les `neuShadow` en light pour plus de profondeur
Dans `neuShadowLight`, changer les `borderColor` pour qu'ils soient légèrement plus visibles :
```ts
elevated:   borderColor: '#f5f8fc',  // quasi blanc mais distinct
elevatedSm: borderColor: '#f0f4f8',
pressed:    borderColor: '#c8d0da',
```

## Contraintes
- Ne PAS changer les clés existantes — ajouter uniquement `input`
- `lightColors` doit avoir TOUTES les clés de `colors` (et vice-versa) — le type `ThemeColors` le garantit
- Ne pas toucher `neuShadowParams` ni `intensityColors`
- Ne pas modifier les valeurs `primaryGradientStart/End` (les gradients de boutons restent identiques)

## Critères de validation
- `npx tsc --noEmit` → zéro erreur (vérifier que `lightColors as ThemeColors` ne cause plus d'erreur de type)
- Les deux palettes ont exactement les mêmes clés
- Visuellement : les cartes ressortent du fond dans les deux modes
- En light mode : le primary `#006d6b` a un ratio de contraste ≥ 4.5:1 sur fond `#eef1f5`
