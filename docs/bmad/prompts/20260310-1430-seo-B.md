# Rapport — SEO contenu & on-page — Groupe B — 20260310-1430

## Objectif
Améliorer le contenu textuel du site pour cibler les mots-clés long-tail FR et ajouter une section FAQ pour les featured snippets Google.

## Fichiers concernés
- `web/src/components/sections/HeroSection.tsx` (H1, subtitle)
- `web/src/components/sections/FeaturesSection.tsx` (H2, descriptions)
- `web/src/data/features.ts` (textes des features)
- `web/src/components/sections/FooterSection.tsx` (ajouter texte SEO)
- **NOUVEAU** `web/src/components/sections/FAQSection.tsx` (section FAQ)
- `web/src/app/page.tsx` (intégrer FAQSection)

## Contexte technique
- Components React (tsx), certains "use client", d'autres server components
- Tailwind CSS pour le styling
- Design neumorphique existant — respecter les classes `shadow-neu-out`, `shadow-neu-in`, `var(--accent)`, etc.
- Les sections utilisent les classes `reveal` pour le scroll animation

## Étapes

### 1. Optimiser le H1 dans HeroSection.tsx
Le H1 actuel "FORGE TON CORPS/PHYSIQUE/..." ne contient aucun mot-clé SEO.

Changer le subtitle (le `<p>` sous le H1) pour inclure des mots-clés :
```
"L'application de musculation gratuite et 100% offline. Crée tes programmes, enregistre chaque série et progresse séance après séance — sans connexion."
```

Ne PAS toucher l'animation du H1 "FORGE TON" — c'est le branding. Mais ajouter un `<span className="sr-only">` avec du texte SEO caché pour les moteurs :
```tsx
<span className="sr-only">Kore — Application de musculation gratuite et tracker d'entraînement offline pour Android</span>
```

### 2. Optimiser le H2 de FeaturesSection.tsx
Changer "Tout ce qu'il te faut" → "Ton tracker de musculation complet"
Changer "Et rien de plus." → "Tout pour suivre ta progression, sans connexion."

### 3. Enrichir les descriptions des features (`features.ts`)
Ajouter des mots-clés naturels dans chaque description. Par exemple :
- "Programmes sur mesure" → ajouter "programme de musculation personnalisé"
- "100% Offline" → ajouter "appli musculation sans internet"
- "Suivi de performance" → ajouter "tracker de musculation"

Garder le ton actuel, juste glisser les mots-clés de manière naturelle.

### 4. Créer une section FAQ (`FAQSection.tsx`)
Ajouter 5-6 questions fréquentes qui ciblent les recherches Google :

1. **"Kore est-elle vraiment gratuite ?"** — Oui, Kore est une application de musculation 100% gratuite. Le plan de base te donne accès à la création de programmes, au suivi de tes séances et à l'historique complet de tes entraînements.

2. **"Est-ce que Kore fonctionne sans connexion internet ?"** — Oui, Kore est un tracker de musculation 100% offline. Toutes tes données sont stockées en local sur ton téléphone. Pas besoin de wifi à la salle de sport.

3. **"Sur quels appareils Kore est-elle disponible ?"** — Kore est disponible sur Android. Une version iOS est prévue pour plus tard.

4. **"Comment créer un programme de musculation avec Kore ?"** — Tu peux créer tes propres programmes personnalisés avec autant de séances que tu veux. Organise tes exercices par muscle ciblé, type d'équipement et objectif.

5. **"Mes données d'entraînement sont-elles sécurisées ?"** — Toutes tes données restent sur ton appareil. Kore ne collecte aucune donnée personnelle et fonctionne sans compte.

6. **"Comment suivre ma progression musculaire ?"** — Kore enregistre chaque série, chaque rep et chaque poids. Tu peux visualiser ta progression avec des graphiques détaillés et retrouver tout ton historique d'entraînement.

Style : même esthétique neumorphique que les features. Utiliser `<details>/<summary>` ou un accordion custom. Ajouter le JSON-LD `FAQPage` dans cette section.

### 5. Intégrer le FAQ dans page.tsx
Ajouter `<FAQSection />` entre PricingSection et SubscribeSection.

### 6. Ajouter un paragraphe SEO dans FooterSection
Un petit paragraphe descriptif avant les liens du footer :
```
"Kore est une application de musculation gratuite pour Android. Tracker d'entraînement 100% offline, elle te permet de créer des programmes personnalisés, suivre tes performances et progresser à chaque séance."
```

## Contraintes

- Respecter le design neumorphique existant
- Garder le ton tutoiement (tu/ton/tes)
- Les textes doivent rester naturels, pas du keyword stuffing
- Ne pas casser les animations scroll reveal
- Le FAQ doit avoir un JSON-LD `FAQPage` pour les rich snippets Google
