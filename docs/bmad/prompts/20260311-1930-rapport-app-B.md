# Rapport — Rapports hebdo/mensuels in-app — Groupe B — 20260311-1930

## Objectif
Ajouter une card "Rapport hebdomadaire" sur le HomeScreen dashboard, entre le Weekly Activity card et les grilles de navigation. La card affiche un résumé compact de la semaine et permet de naviguer vers l'écran détail.

## Fichiers concernés
- `mobile/src/components/WeeklyReportCard.tsx` (NOUVEAU)
- `mobile/src/screens/HomeScreen.tsx` (ajouter la card)

## Contexte technique

### HomeScreen actuel (layout dans ScrollView, de haut en bas)
1. Header Card — greeting + 3 KPIs + settings
2. Gamification Card — level, XP, streak, badges
3. Quick-Start Card (conditionnel) — dernière séance + "Go"
4. Weekly Activity Card (`weeklyCard`) — bande 7 jours
5. **← INSÉRER ICI : WeeklyReportCard**
6. Section grids — Training (Programs, Exercises, Calendar) + Stats (Duration, Volume, Measures)

### Données déjà observées dans HomeScreen via `withObservables`
- `histories` : 90 derniers jours, non-deleted, non-abandoned
- `sets` : 30 derniers jours
- `sessions` : tous

### Pattern UI existant
- Cards avec `colors.card` background, `borderRadius.md` (14), `spacing.md` padding
- Icônes avec `Ionicons` (de `@expo/vector-icons`)
- Navigation : `navigation.navigate('ReportDetail')` (le screen sera ajouté par le Groupe C)
- Thème dynamique : `useColors()` hook
- i18n : `useLanguage()` hook → `{ t }`
- Haptics : `useHaptics()` hook

### Design de la card
```
┌─────────────────────────────────────┐
│ 📊  Rapport de la semaine     Voir →│
│                                     │
│  4 séances   2 450 kg   2 PRs      │
│  ↑12% vs sem. dernière             │
│                                     │
│  Top muscles : Pectoraux, Dos, ...  │
└─────────────────────────────────────┘
```

- Titre : `t.weeklyReport` (FR: "Rapport de la semaine")
- KPIs en ligne : sessions, volume (formatVolume), PRs
- Comparaison % vs semaine précédente (vert si +, rouge si -)
- Top 3 muscles sous forme de chips/texte
- Tap → navigate vers `ReportDetail`

### Clés i18n à ajouter
Dans `mobile/src/i18n/fr.ts` et `en.ts` :
```
weeklyReport: "Rapport de la semaine" / "Weekly Report"
vsLastWeek: "vs sem. dernière" / "vs last week"
topMuscles: "Top muscles" / "Top muscles"
viewReport: "Voir" / "View"
```

## Étapes

1. Ajouter les clés i18n dans `fr.ts` et `en.ts`

2. Créer `WeeklyReportCard.tsx` :
   - Props : `sessionsCount`, `totalVolumeKg`, `prsCount`, `comparedToPrevious`, `topMuscles`, `onPress`
   - Design neumorphique cohérent avec les autres cards HomeScreen
   - Utiliser `useColors()`, `useLanguage()`, `useHaptics()`

3. Intégrer dans `HomeScreen.tsx` :
   - Importer `WeeklyReportCard`
   - Calculer les données avec `computeReportSummary` (du Groupe A) + `getWeekPeriod(0)`
   - OU temporairement : calculer directement avec les helpers existants si le Groupe A n'est pas encore fait
   - Placer après le Weekly Activity Card

- Respecter le pattern `withObservables` — pas de `useState` pour les données DB
- La card doit fonctionner même si le Groupe A n'est pas encore implémenté (utiliser les helpers existants directement en fallback)
- Pas de nouvelle dépendance npm
- Haptic feedback sur le tap (`haptics.onPress()`)
