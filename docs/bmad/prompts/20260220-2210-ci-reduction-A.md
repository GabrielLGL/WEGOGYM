# Rapport — ci-reduction — Groupe A — 20260220-2210

## Objectif
Ajouter des filtres `paths-ignore` dans les deux workflows GitHub Actions pour éviter que les commits docs-only (rapports bmad, review, git-history, etc.) déclenchent inutilement la CI. Actuellement 214 runs pour des commits de documentation pure.

## Fichiers concernés
- `.github/workflows/ci.yml`
- `.github/workflows/lint.yml`

## Contexte technique
- Stack : GitHub Actions, React Native Expo 52, monorepo avec `mobile/` et `docs/`
- Les commits de type `docs(*)` représentent ~60% des push sur main (rapports bmad, reviews, git-history)
- Ces commits ne touchent JAMAIS `mobile/src/` ni `.github/`
- Les deux workflows ont la même structure `on: push / pull_request branches: [main, develop]`
- Il n'y a PAS de `paths` filter actuellement

## Étapes

1. Lire `.github/workflows/ci.yml` et `.github/workflows/lint.yml` en entier

2. Dans **ci.yml** : ajouter `paths-ignore` sous `on.push` ET `on.pull_request` :
```yaml
on:
  push:
    branches: [main, develop]
    paths-ignore:
      - 'docs/**'
      - '*.md'
      - '.gitignore'
  pull_request:
    branches: [main, develop]
    paths-ignore:
      - 'docs/**'
      - '*.md'
      - '.gitignore'
```

3. Faire la même modification dans **lint.yml** (même structure)

4. Vérifier que les fichiers YAML sont valides (indentation, pas de tabs)

5. Ne PAS modifier les étapes `jobs:` ni les `steps:` — uniquement les triggers `on:`

## Contraintes
- Ne pas casser : la CI doit toujours se déclencher sur les vrais changements code (`mobile/src/`, `mobile/package.json`, `.github/workflows/`)
- Ne pas toucher : jobs, steps, matrix, cache config
- Les `paths-ignore` doivent être sur BOTH push et pull_request
- YAML : indentation 2 espaces, pas de tabs

## Critères de validation
- `cat .github/workflows/ci.yml` → `paths-ignore` visible sous `push:` et `pull_request:`
- `cat .github/workflows/lint.yml` → même chose
- YAML syntaxiquement valide (pas d'erreur d'indentation)
- Un commit docs-only (ex: `git commit -m "docs: test"` sur un fichier `docs/test.md`) ne devrait plus déclencher la CI
