# Rapport — Audit 6 Site Kore — Groupe B : Privacy Page Accessibilité — 20260227-2000

## Objectif
5 corrections d'accessibilité dans `privacy/page.tsx` :
1. Ajouter un **skip link** (lien d'évitement) pour la navigation clavier
2. Ajouter `id="main-content"` sur `<main>` (cible du skip link)
3. Ajouter `<caption>` sur le `<table>` des données collectées
4. Ajouter `aria-hidden="true"` sur la flèche du lien retour **bas de page** (ligne 339) — déjà fait en haut (ligne 65), manquant en bas
5. Ajouter `aria-label` sur les liens externes vers les politiques de confidentialité des sous-traitants

## Fichiers concernés
- `web/src/app/privacy/page.tsx`

## Contexte technique
- Page Next.js App Router (Server Component pur — pas de `"use client"`)
- Utilise des CSS custom properties (`var(--accent)`, `var(--bg)`, etc.)
- Le skip link doit utiliser les mêmes classes que la page d'accueil (`page.tsx`) pour la cohérence
- `npx tsc --noEmit` (depuis `web/`) doit rester à 0 erreur

## Étapes

### 1. Ajouter le skip link (avant `<BackgroundBlobs />`, ligne ~52)

Insérer immédiatement après `<div className="min-h-screen relative overflow-hidden">` et avant `<BackgroundBlobs />` :

```tsx
<div className="min-h-screen relative overflow-hidden">
  {/* Skip link — navigation clavier */}
  <a
    href="#main-content"
    className="sr-only focus:not-sr-only focus:fixed focus:top-4 focus:left-4 focus:z-[200] focus:bg-[var(--accent)] focus:text-white focus:px-4 focus:py-2 focus:rounded-lg focus:text-sm focus:font-semibold"
  >
    Aller au contenu principal
  </a>
  <BackgroundBlobs />
```

### 2. Ajouter `id="main-content"` sur `<main>` (ligne 56)

```tsx
// AVANT
<main className="relative z-[2] py-20 px-6">

// APRÈS
<main id="main-content" className="relative z-[2] py-20 px-6">
```

### 3. Ajouter `<caption>` sur le `<table>` (ligne ~124)

Immédiatement après `<table className="w-full text-sm text-[var(--text-muted)] border-collapse">`, avant `<thead>` :

```tsx
<table className="w-full text-sm text-[var(--text-muted)] border-collapse">
  <caption className="sr-only">Données personnelles collectées</caption>
  <thead>
```

### 4. Ajouter `aria-hidden="true"` sur la flèche bas de page (ligne 339)

```tsx
// AVANT
<span className="group-hover:-translate-x-1 transition-transform inline-block">
  ←
</span>

// APRÈS
<span aria-hidden="true" className="group-hover:-translate-x-1 transition-transform inline-block">
  ←
</span>
```

Note : le lien retour en HAUT de page (ligne 65) a déjà `aria-hidden="true"` — c'est uniquement le lien bas de page (ligne 339) qui manque cet attribut.

### 5. Ajouter `aria-label` sur les liens externes sous-traitants (ligne ~220)

```tsx
// AVANT
<a
  href={provider.url}
  target="_blank"
  rel="noopener noreferrer"
  className="text-[var(--accent)] text-xs hover:underline"
>
  Politique de confidentialité → {provider.label}
</a>

// APRÈS
<a
  href={provider.url}
  target="_blank"
  rel="noopener noreferrer"
  aria-label={`Politique de confidentialité de ${provider.label} (nouvel onglet)`}
  className="text-[var(--accent)] text-xs hover:underline"
>
  Politique de confidentialité → {provider.label}
</a>
```

## Contraintes
- Ne modifier QUE les éléments listés ci-dessus
- Ne pas modifier les styles, la structure JSX, les composants `Section`, `BulletList`
- `className="sr-only"` est une classe Tailwind CSS standard (déjà présente dans le projet)
- Garder le `&apos;` là où il existe déjà

## Critères de validation
- `npx tsc --noEmit` (depuis `web/`) → 0 erreur
- Navigation Tab depuis la barre d'adresse : le skip link apparaît en premier, visible en violet
- Lecteur d'écran : table annoncée avec le nom "Données personnelles collectées"
- Lecteur d'écran : flèche du lien retour bas de page non annoncée
- Lecteur d'écran : liens sous-traitants annoncés avec "(nouvel onglet)"
