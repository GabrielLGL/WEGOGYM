## Implémentation

### Structure de données (calcul dans useMemo)
```ts
interface BalanceData {
  pushVolume: number      // Σ(weight*reps) pour muscles push
  pullVolume: number      // Σ(weight*reps) pour muscles pull
  legsVolume: number      // Σ(weight*reps) pour muscles legs
  pushPullRatio: number   // pushVolume / pullVolume (1.0 = équilibré)
  upperLowerRatio: number // (push+pull) / legs (idéal ~1.5-2.0)
  pushPct: number         // % push sur total push+pull+legs
  pullPct: number         // % pull sur total push+pull+legs
  legsPct: number         // % legs sur total push+pull+legs
  totalSets: number       // nombre total de sets considérés
}
```

**Algorithme (dans useMemo) :**
1. Construire `Map<exerciseId, muscles[]>` depuis exercises
2. Pour chaque set : `volume = set.weight * set.reps`
3. Classer le volume dans push/pull/legs selon les muscles de l'exercice
   - Un exercice peut toucher plusieurs catégories → diviser le volume équitablement
4. Calculer ratios :
   - `pushPullRatio = pushVolume / Math.max(1, pullVolume)`
   - `upperLowerRatio = (pushVolume + pullVolume) / Math.max(1, legsVolume)`
5. Retourner `null` si `totalSets < 10`

### Zones d'équilibre
```ts
function getPushPullStatus(ratio: number): 'balanced' | 'push_dominant' | 'pull_dominant' {
  if (ratio > 1.5) return 'push_dominant'
  if (ratio < 0.67) return 'pull_dominant'
  return 'balanced'
}

function getUpperLowerStatus(ratio: number): 'balanced' | 'upper_dominant' | 'legs_dominant' {
  if (ratio > 3.0) return 'upper_dominant'
  if (ratio < 0.8) return 'legs_dominant'
  return 'balanced'
}
```
