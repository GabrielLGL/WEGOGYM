## Contexte technique
- Jest + React Native Testing Library (`@testing-library/react-native`)
- Chaque ecran a une version `*Base` non-wrapped par withObservables — c'est celle a tester
- Pattern : voir `StatsConstellationScreen.test.tsx` comme reference

## Pattern de test

```tsx
import React from 'react'
import { render } from '@testing-library/react-native'
import { SomeScreenBase } from '../SomeScreen'

jest.mock('expo-haptics', () => ({
  impactAsync: jest.fn(),
  notificationAsync: jest.fn(),
  ImpactFeedbackStyle: { Light: 'Light', Medium: 'Medium', Heavy: 'Heavy' },
  NotificationFeedbackType: { Success: 'Success', Error: 'Error' },
}))

jest.mock('../../model/index', () => ({
  database: { write: jest.fn(), get: jest.fn() },
}))
```

## Consignes pour chaque ecran

### Regles generales
1. **Lire le fichier source** avant d'ecrire le test
2. Importer la version `*Base` (ou le composant exporte si pas de withObservables)
3. Minimum 3 tests : vide, empty state, avec donnees
4. Mock `expo-haptics` et `../../model/index`
5. Mock navigation si utilise : `jest.mock('@react-navigation/native', () => ({ useNavigation: () => ({ navigate: jest.fn(), goBack: jest.fn() }), useRoute: () => ({ params: {} }) }))`

### ExerciseCollectionScreen
- Rendu vide (pas d'exercices)
- Affiche la collection avec exercices
- Filtre par muscle/equipement

### ExerciseCardScreen
- Rendu avec un exercice
- Affiche le nom, muscles, stats
- Gere l'absence de stats (nouvel exercice)

### ProgressPhotosScreen
- Rendu vide (pas de photos)
- Mock `expo-image-picker` si utilise : `jest.mock('expo-image-picker', () => ({ launchCameraAsync: jest.fn(), launchImageLibraryAsync: jest.fn(), MediaTypeOptions: { Images: 'Images' } }))`
- Affiche la grille avec photos mock

### ReportDetailScreen
- Rendu avec un rapport
- Affiche les KPIs du rapport
- Mock route params avec un rapport de test

### LegalScreen
- Rendu sans props
- Affiche le texte CGU/politique
- Composant le plus simple — peut etre un simple snapshot test

## Contraintes
- NE PAS modifier les ecrans
- Minimum 3 tests par ecran (sauf LegalScreen : 2 suffisent)
- Noms de tests en francais
- Tous les tests doivent passer
- Mock les services natifs si necessaire (image-picker, camera, etc.)
