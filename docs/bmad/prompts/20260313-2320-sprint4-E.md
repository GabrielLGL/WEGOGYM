## Implémentation

### Structure d'une "carte d'activité"
```ts
interface ActivityCard {
  historyId: string
  sessionName: string              // session.name ou fallback "Séance"
  startTime: Date
  durationMin: number | null       // null si endTime === null
  totalVolumeKg: number            // Σ(weight * reps) des sets
  totalSets: number                // count des sets
  totalPrs: number                 // count des sets avec isPr === true
  topExercises: string[]           // 3 premiers noms d'exercices distincts
  topMuscles: string[]             // 3 premiers groupes musculaires distincts
  isAbandoned: boolean
}
```

### Logique de construction (dans useMemo)
1. Filtrer `histories` : `deletedAt === null` (conserver `isAbandoned` pour affichage)
2. Trier par `startTime` décroissant (plus récent en premier)
3. Limiter à 50 entrées (pagination différée via `FlatList.onEndReached`)
4. Pour chaque history, agréger les sets correspondants (utiliser un `Map<historyId, Set[]>` pré-construit)
5. Construire `ActivityCard` pour chaque history

**Format date relative :**
```ts
function formatRelativeDate(date: Date, t: ...): string {
  const now = Date.now()
  const diffMs = now - date.getTime()
  const diffDays = Math.floor(diffMs / (24 * 60 * 60 * 1000))
  if (diffDays === 0) return t.activityFeed.today
  if (diffDays === 1) return t.activityFeed.yesterday
  if (diffDays < 7) return t.activityFeed.daysAgo.replace('{n}', String(diffDays))
  // Au-delà de 7j : format "12 fév" ou "12 Feb"
  return date.toLocaleDateString(language === 'fr' ? 'fr-FR' : 'en-US', { day: 'numeric', month: 'short' })
}
```

### withObservables pattern
```ts
const enhance = withObservables([], () => ({
  sets: database.get<WorkoutSet>('sets').query(
    Q.on('histories', Q.where('deleted_at', null))
  ).observe(),
  exercises: database.get<Exercise>('exercises').query().observe(),
  histories: database.get<History>('histories').query(
    Q.where('deleted_at', null),
    Q.sortBy('start_time', Q.desc),
  ).observe(),
  sessions: database.get<Session>('sessions').query().observe(),
}))
```

## Contraintes
- Tap sur une carte → `navigation.navigate('HistoryDetail', { historyId: card.historyId })` (route déjà existante)
- Utiliser `FlatList` (pas `ScrollView`) pour la performance sur large jeux de données
- `initialNumToRender={10}` sur la FlatList
- Pas de tri côté JS — utiliser le tri SQL `Q.sortBy('start_time', Q.desc)` dans la query
- Pas de dépendances npm supplémentaires
- Respecter `spacing.*`, `borderRadius.*`, `fontSize.*` du thème
- Cards : `backgroundColor: colors.card`, pas de border (ou `borderWidth: 1, borderColor: colors.separator`)

## Critères de validation
- `npx tsc --noEmit` → zéro erreur TypeScript
- La tuile apparaît sur HomeScreen section "Entraînement"
- Les cartes s'affichent correctement pour un utilisateur avec des séances
- Vide state affiché si aucune séance
- Tap → navigation vers HistoryDetail
