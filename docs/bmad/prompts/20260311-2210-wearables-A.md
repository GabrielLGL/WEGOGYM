# Rapport — Intégration wearables — Groupe A — 20260311-2210

## Objectif
Étendre le schéma WatermelonDB (v37 → v38) pour supporter les préférences de synchronisation wearables : champs dans `users` + table `wearable_sync_logs` pour l'historique de synchro.

## Fichiers concernés
- `mobile/src/model/schema.ts` — incrémenter v37 → v38, ajouter colonnes `users`, ajouter table `wearable_sync_logs`
- `mobile/src/model/migrations.ts` — ajouter migration v37 → v38
- `mobile/src/model/models/WearableSyncLog.ts` — **nouveau fichier** : modèle WatermelonDB
- `mobile/src/model/models/User.ts` — ajouter nouveaux champs wearable
- `mobile/src/model/index.ts` — enregistrer `WearableSyncLog`

## Contexte technique

### Stack
- WatermelonDB (SQLite/JSI) — schéma actuel **v37**
- TypeScript strict, `experimentalDecorators: true`, `useDefineForClassFields: false`
- `User` est une table single-row (préférences utilisateur globales)
- Mutations DB obligatoirement dans `database.write()`

### Nouveaux champs dans `users`
| Colonne | Type | Optional | Description |
|---------|------|----------|-------------|
| `wearable_provider` | `string` | oui | `'health_connect'` \| `'healthkit'` \| `null` |
| `wearable_sync_weight` | `boolean` | non | Synchro poids auto depuis wearable (default: false) |
| `wearable_last_sync_at` | `number` | oui | Timestamp de la dernière synchro réussie |

### Nouvelle table `wearable_sync_logs`
| Colonne | Type | Optional | Description |
|---------|------|----------|-------------|
| `sync_at` | `number` | non | Timestamp de la synchro |
| `provider` | `string` | non | `'health_connect'` \| `'healthkit'` |
| `status` | `string` | non | `'success'` \| `'error'` \| `'partial'` |
| `records_synced` | `number` | oui | Nombre d'enregistrements importés |
| `error_message` | `string` | oui | Message d'erreur si status = 'error' |
| `created_at` | `number` | non | Timestamp création |
| `updated_at` | `number` | non | Timestamp mise à jour |

### Patterns WatermelonDB à respecter
- `@field()` pour les types number/string/boolean
- `@readonly @date('created_at')` et `@readonly @date('updated_at')`
- Ajouter colonnes `boolean` avec `isOptional: false` et `default: false`
- Migration via `addColumns` pour `users`, `createTable` pour la nouvelle table

### Pattern User.ts existant
Voir `mobile/src/model/models/User.ts` — la classe a ~68 lignes, fields déclarés avec `@field()`.
Ajouter les 3 nouveaux champs à la suite des existants.

### Pattern schema.ts existant
Voir `mobile/src/model/schema.ts` — chaque table est un `tableSchema({...})`.
Pour `boolean` : `{ name: 'wearable_sync_weight', type: 'boolean', isOptional: false }`.

## Étapes

1. **Modifier** `mobile/src/model/models/User.ts`
   - Ajouter `@field('wearable_provider') wearableProvider!: string | null`
   - Ajouter `@field('wearable_sync_weight') wearableSyncWeight!: boolean`
   - Ajouter `@field('wearable_last_sync_at') wearableLastSyncAt!: number | null`

2. **Créer** `mobile/src/model/models/WearableSyncLog.ts`
   - Classe `WearableSyncLog extends Model`
   - `static table = 'wearable_sync_logs'`
   - Decorators pour chaque colonne (cf tableau ci-dessus)

3. **Modifier** `mobile/src/model/schema.ts`
   - Incrémenter `version` de 37 à 38
   - Ajouter les 3 colonnes dans la table `users`
   - Ajouter `tableSchema({ name: 'wearable_sync_logs', columns: [...] })`

4. **Modifier** `mobile/src/model/migrations.ts`
   - Ajouter migration `{ toVersion: 38, steps: [addColumns('users', [...]), createTable('wearable_sync_logs', [...])] }`

5. **Modifier** `mobile/src/model/index.ts`
   - Importer `WearableSyncLog`
   - Ajouter au tableau `modelClasses`

## Contraintes
- Ne pas toucher aux tables existantes autres que `users`
- Schema ↔ Model sync : chaque `@field` doit avoir sa colonne et vice versa
- Pas de `any` TypeScript
- Pas de `console.log` sans `__DEV__`
- Les credentials wearable (tokens OAuth) → jamais en SQLite → `expo-secure-store`

## Critères de validation
- `npx tsc --noEmit` → zéro erreur
- `npm test` → zéro fail
- Le schéma v38 compile et la migration est cohérente avec v37
- `WearableSyncLog` a tous les decorators correspondant aux colonnes schema
