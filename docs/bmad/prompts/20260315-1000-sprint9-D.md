# Rapport — Standards de Force (#94) — Groupe D — 20260315-1000

## Objectif
Créer un écran `StatsStrengthScreen` comparant les 1RM estimés de l'utilisateur sur les exercices principaux (squat, bench press, deadlift, etc.) à des benchmarks de force par catégorie de poids corporel. L'écran affiche le niveau (Débutant → Elite) pour chaque lift. Accessible depuis StatsScreen.

- `mobile/src/screens/StatsStrengthScreen.tsx` — NOUVEAU : écran principal

## Contexte technique

### Stack & Architecture
- React Native + Expo 52 + TypeScript strict (Fabric / New Architecture)
- WatermelonDB — données via `withObservables`
- `useColors()` / `useLanguage()` / `useDeferredMount()`
- `EPLEY_FORMULA_DIVISOR` depuis `mobile/src/model/constants.ts` (= 30)
- Formule 1RM Epley : `weight * (1 + reps / EPLEY_FORMULA_DIVISOR)`

## Implémentation

### strengthStandardsHelpers.ts

```ts
export type StrengthLevel = 'beginner' | 'novice' | 'intermediate' | 'advanced' | 'elite'

export interface StrengthBenchmark {
  exerciseName: string            // nom canonique (ex: "Développé couché")
  exercisePatterns: string[]      // patterns pour matcher les noms d'exercice en DB
  // Multiplier du poids corporel par niveau (homme, approximatif)
  standards: Record<StrengthLevel, number>  // ex: { beginner: 0.5, novice: 0.75, ... }
}

export interface StrengthResult {
  exerciseName: string
  matchedExerciseId: string | null
  estimated1RM: number | null     // null si pas de données
  bodyweightRatio: number | null  // 1RM / poids corporel
  level: StrengthLevel | null
  nextLevelThreshold: number | null  // poids à atteindre pour le niveau suivant
}

/**
 * Benchmarks de force basés sur des standards reconnus (simplifié).
 * Ratios poids corporel pour homme adulte.
 */
export const STRENGTH_BENCHMARKS: StrengthBenchmark[] = [
  {
    exerciseName: 'Développé couché',
    exercisePatterns: ['développé couché', 'bench press', 'dev couché', 'dc barre'],
    standards: { beginner: 0.5, novice: 0.75, intermediate: 1.0, advanced: 1.25, elite: 1.5 },
  },
  {
    exerciseName: 'Squat',
    exercisePatterns: ['squat', 'squat barre', 'back squat'],
    standards: { beginner: 0.75, novice: 1.0, intermediate: 1.25, advanced: 1.75, elite: 2.25 },
  },
  {
    exerciseName: 'Soulevé de terre',
    exercisePatterns: ['soulevé de terre', 'deadlift', 'sdt'],
    standards: { beginner: 1.0, novice: 1.25, intermediate: 1.5, advanced: 2.0, elite: 2.75 },
  },
  {
    exerciseName: 'Développé militaire',
    exercisePatterns: ['développé militaire', 'overhead press', 'ohp', 'dev militaire'],
    standards: { beginner: 0.35, novice: 0.5, intermediate: 0.65, advanced: 0.85, elite: 1.1 },
  },
  {
    exerciseName: 'Rowing barre',
    exercisePatterns: ['rowing barre', 'barbell row', 'rowing'],
    standards: { beginner: 0.5, novice: 0.65, intermediate: 0.85, advanced: 1.1, elite: 1.4 },
  },
]

export function matchExercise(
  benchmark: StrengthBenchmark,
  exercises: Array<{ id: string; name: string }>,
): string | null  // retourne exerciseId ou null

export function computeStrengthStandards(
  exercises: Array<{ id: string; name: string }>,
  sets: Array<{ weight: number; reps: number; exerciseId: string }>,
  bodyweight: number | null,
): StrengthResult[]
```

**Algorithme matchExercise :**
- Pour chaque pattern, vérifier si `exercise.name.toLowerCase().includes(pattern)`
- Retourner le premier match

**Algorithme computeStrengthStandards :**
1. Pour chaque benchmark, trouver l'exercice correspondant via `matchExercise`
2. Si match trouvé : calculer le 1RM max (Epley) depuis tous les sets de cet exercice
3. Si bodyweight disponible : calculer le ratio et déterminer le niveau
4. `nextLevelThreshold` = bodyweight × standards[nextLevel]

### withObservables pattern
```ts
const enhance = withObservables([], () => ({
  user: observeCurrentUser(),
  sets: database.get<WorkoutSet>('sets').query(
    Q.on('histories', Q.and(
      Q.where('deleted_at', null),
      Q.or(Q.where('is_abandoned', null), Q.where('is_abandoned', false)),
    )),
  ).observe(),
  exercises: database.get<Exercise>('exercises').query().observe(),
  measurements: database.get<BodyMeasurement>('body_measurements').query(
    Q.sortBy('date', Q.desc),
    Q.take(1),
  ).observe(),
}))
```

## Contraintes
- Pas de migration DB
- `useDeferredMount()` comme wrapper
- Pattern matching case-insensitive pour les noms d'exercice
- Si aucun poids corporel → afficher message mais permettre quand même de voir les 1RM
- `observeCurrentUser()` depuis `databaseHelpers.ts`

## Critères de validation
- `npx tsc --noEmit` → zéro erreur
- Bouton "Force" dans StatsScreen
- 5 lifts affichés avec niveaux corrects
- Fonctionne sans poids corporel (affiche juste les 1RM)
- Pattern matching trouve les exercices existants
