# Prompt — Deload / Périodisation (recommandations repos) — 20260311-2000

## Demande originale
`FINAL` Deload / périodisation (recommandations repos)

## Description
Moteur de détection de fatigue + UI de recommandation sur le HomeScreen.
Analyse les données d'entraînement existantes (histories, sets, user) pour détecter :
- Jours consécutifs d'entraînement excessifs
- Spike de volume par rapport à la moyenne
- Block d'entraînement long sans pause
- Surcharge musculaire (sets > MRV)
- Manque de jours de repos

Affiche une card conditionnelle sur le HomeScreen avec la recommandation prioritaire.

## Décisions architecturales
- **Stateless** : calcul à la volée depuis les données existantes (pas de nouvelle table)
- **Pas de schema change** (v35 maintenue) — `deload_acknowledged_week` reporté en v2
- **Streak non modifié** — le streak reflète la réalité, message rassurant dans la card
- **Dismiss session-scoped** — `useState` local, réapparaît au relancement
- **MRV thresholds** — basés sur Renaissance Periodization, ajustés par `userLevel`
