# Rapport — Rapports hebdo/mensuels in-app — Groupe A — 20260311-1930

## Objectif
Créer un helper `statsReport.ts` qui agrège les fonctions stats existantes pour produire un objet `WeeklyReport` / `MonthlyReport` prêt à afficher. Pas de nouveau modèle DB — tout est calculé à la volée depuis les données réactives existantes.

## Fichiers concernés
- `mobile/src/model/utils/statsReport.ts` (NOUVEAU)
- `mobile/src/model/utils/statsTypes.ts` (ajouter les interfaces Report)
- `mobile/src/model/utils/statsHelpers.ts` (re-exporter le nouveau module)

## Contexte technique

### Fonctions existantes à réutiliser (NE PAS réécrire)
Toutes dans `mobile/src/model/utils/` :
- `computeGlobalKPIs(histories, sets, ctx?)` → `{ totalSessions, totalVolumeKg, totalPRs }`
- `computeVolumeStats(sets, exercises, histories, period, ctx?)` → `{ total, perWeek, topExercises, comparedToPrevious }`
- `computeMuscleRepartition(sets, exercises, histories, period, othersLabel)` → `MuscleRepartitionEntry[]`
- `computePRsByExercise(sets, exercises, histories, ctx?)` → `ExercisePR[]`
- `computeDurationStats(histories)` → `{ avgMin, totalHours, ... }`
- `computeCurrentStreak(histories)` → `number`
- `computeTopExercisesByFrequency(sets, exercises, histories, limit?, ctx?)` → `ExerciseFrequency[]`
- `prepareStatsContext(histories, exercises)` → `StatsContext` (appeler UNE fois, passer à toutes les fonctions)
- `buildWeeklyActivity(histories, sets, sessions, dayLabels?)` → `WeeklyActivityData`
- `formatVolume(kg)` → string
- `formatDuration(min)` → string

### Pattern clé
```typescript
const ctx = prepareStatsContext(histories, exercises)
// Puis passer ctx à chaque compute*() pour éviter les rebuilds O(N)
```

### Interfaces à créer dans `statsTypes.ts`
```typescript
interface ReportPeriod {
  type: 'weekly' | 'monthly'
  startDate: number  // timestamp ms
  endDate: number    // timestamp ms
  label: string      // ex: "Sem. 10 — Mars 2026" ou "Mars 2026"
}

interface ReportSummary {
  period: ReportPeriod
  sessionsCount: number
  totalVolumeKg: number
  totalDurationMin: number
  avgDurationMin: number
  prsCount: number
  currentStreak: number
  comparedToPrevious: number  // % volume vs période précédente
  topMuscles: MuscleRepartitionEntry[]  // top 3
  topExercises: VolumeTopExercise[]     // top 3
  prs: ExercisePR[]                     // PRs de la période
}
```

## Étapes

1. Ajouter `ReportPeriod` et `ReportSummary` dans `statsTypes.ts`

2. Créer `statsReport.ts` avec :
   ```typescript
   function getWeekPeriod(weekOffset: number): ReportPeriod
   // weekOffset=0 = semaine courante, -1 = semaine dernière, etc.

   function getMonthPeriod(monthOffset: number): ReportPeriod
   // monthOffset=0 = mois courant, -1 = mois dernier, etc.

   function computeReportSummary(
     histories: History[],
     sets: Set[],
     exercises: Exercise[],
     period: ReportPeriod,
     ctx?: StatsContext
   ): ReportSummary
   // Filtre histories/sets à la période, puis appelle les compute*() existants
   ```

3. Re-exporter depuis `statsHelpers.ts`

## Contraintes
- Ne pas modifier les fonctions stats existantes
- Ne pas créer de nouveau modèle WatermelonDB (pas de schema change)
- Filtrer histories par `deletedAt === null && !isAbandoned` (pattern existant)
- Filtrer histories/sets par date de la période
- TypeScript strict — pas de `any`
- Utiliser les imports existants de `statsHelpers.ts`
