## Logique des actions

### "Connecter"
1. `wearableService?.requestPermissions()` → si refusé : afficher message d'erreur dans une AlertDialog
2. Si accordé : `database.write(() => user.update(u => { u.wearableProvider = WEARABLE_PROVIDER }))`
3. Lancer une première synchro immédiate

### "Déconnecter"
1. AlertDialog "Déconnecter le wearable ?" → confirmer
2. `database.write(() => user.update(u => { u.wearableProvider = null; u.wearableLastSyncAt = null }))`

### "Synchroniser maintenant"
1. `wearableService?.fetchWeightRecords(lastSyncDate, now)`
2. Pour chaque poids récupéré : créer un `BodyMeasurement` si la date n'existe pas déjà
3. Créer un `WearableSyncLog` avec le résultat
4. Mettre à jour `user.wearableLastSyncAt`
5. `haptics.onSuccess()` si tout va bien

### Toggle "Synchro auto du poids"
- `database.write(() => user.update(u => { u.wearableSyncWeight = !u.wearableSyncWeight }))`
- `haptics.onSelect()`

## Contraintes
- **JAMAIS** de `<Modal>` natif — Portal + AlertDialog + BottomSheet uniquement
- Pas de couleurs hardcodées — `useColors()` partout
- Pas de texte hardcodé — `useLanguage()` + i18n
- Pas de `any` TypeScript
- Pas de `console.log` sans `__DEV__`
- Toutes les mutations DB dans `database.write()`
- La section wearable est visible **uniquement si** `wearableService !== null` (i.e. Android ou iOS — pas web)
- Si `wearableService === null` : ne pas afficher la section du tout
