# Rapport — Tests Helpers Lot E — Stabilisation 1-E — 20260318-1800

## Objectif
Ecrire les tests unitaires pour 7 helpers UX/training qui n'ont pas de couverture.
Phase 1 du RAPPORT-MVP : stabilisation avant release.

## Fichiers a creer
- `mobile/src/model/utils/__tests__/exerciseOfWeekHelpers.test.ts`
- `mobile/src/model/utils/__tests__/motivationHelpers.test.ts`
- `mobile/src/model/utils/__tests__/workoutTipsHelpers.test.ts`
- `mobile/src/model/utils/__tests__/variantHelpers.test.ts`
- `mobile/src/model/utils/__tests__/personalChallengesHelpers.test.ts`
- `mobile/src/model/utils/__tests__/trainingSplitHelpers.test.ts`
- `mobile/src/model/utils/__tests__/sessionComparisonHelpers.test.ts`

## Contexte technique
- Jest + TypeScript
- Pas de mock de DB — ces helpers sont des fonctions pures
- Pattern existant : voir `__tests__/fatigueIndexHelpers.test.ts` comme reference
- Helper factory : `makeSet(daysAgo, weight, reps)`, `makeHistory(daysAgo)`

## Consignes pour chaque fichier test

### 1. exerciseOfWeekHelpers.test.ts
Lire `exerciseOfWeekHelpers.ts`, identifier les fonctions exportees.
Tests :
- Retourne null si aucun exercice
- Selection basee sur la semaine en cours (deterministe)
- Meme exercice retourne pour la meme semaine
- Exercice different la semaine suivante
- Inclut les muscles cibles

### 2. motivationHelpers.test.ts
Lire `motivationHelpers.ts`, identifier les fonctions exportees.
Tests :
- Retourne un message meme sans donnees
- Message contextuel si longue absence (>7j)
- Message different si streak actif
- Message pour nouvel utilisateur
- Toujours une string non-vide

### 3. workoutTipsHelpers.test.ts
Lire `workoutTipsHelpers.ts`, identifier les fonctions exportees.
Tests :
- Retourne un tip meme sans donnees
- Tips contextuels selon le pattern d'entrainement
- Pas de doublons consecutifs si possible
- Toujours une string non-vide

### 4. variantHelpers.test.ts
Lire `variantHelpers.ts`, identifier les fonctions exportees.
Tests :
- Retourne vide si exercice inconnu
- Variantes partagent le meme groupe musculaire principal
- L'exercice source n'est pas dans ses variantes
- Resultat coherent pour exercices classiques (bench press, squat, etc.)

### 5. personalChallengesHelpers.test.ts
Lire `personalChallengesHelpers.ts`, identifier les fonctions exportees.
Tests :
- Genere des challenges meme avec peu de donnees
- Challenge a un objectif et une progression
- Progression dans [0, 100]
- Challenges adaptes au niveau de l'utilisateur
- Gere le cas sans historique

### 6. trainingSplitHelpers.test.ts
Lire `trainingSplitHelpers.ts`, identifier les fonctions exportees.
Tests :
- Retourne 'unknown' si pas assez de donnees
- Detecte PPL (Push/Pull/Legs)
- Detecte Upper/Lower
- Detecte Full Body
- Pourcentage de confiance dans [0, 100]

### 7. sessionComparisonHelpers.test.ts
Lire `sessionComparisonHelpers.ts`, identifier les fonctions exportees.
Tests :
- Retourne null si pas de seance precedente
- Comparaison volume correct (delta + pourcentage)
- Comparaison duree correcte
- Gere les sessions avec des exercices differents
- Direction correcte (up/down/same)

## Contraintes

- Noms de tests en francais
- `npm test -- --testPathPattern="exerciseOfWeek|motivation|workoutTips|variant|personalChallenges|trainingSplit|sessionComparison"` doit passer
