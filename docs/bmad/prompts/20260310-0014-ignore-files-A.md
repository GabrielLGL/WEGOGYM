# Rapport — Complétion des fichiers .ignore — Groupe A — 20260310-0014

## Objectif
Corriger et compléter tous les fichiers .ignore du projet Kore pour qu'ils soient exhaustifs et conformes aux bonnes pratiques.

## Fichiers concernés
- `.claudeignore` (racine)
- `mobile/.easignore`
- `mobile/.gitignore`
- `.gitignore` (racine)
- `.vercelignore` (racine)

## Contexte technique
- Projet monorepo : `mobile/` (React Native/Expo 52) + `web/` (Next.js)
- EAS Build pour Android (`.easignore` contrôle ce qui est uploadé au build cloud)
- Vercel pour le déploiement web (`.vercelignore` contrôle ce qui est uploadé)
- `.claudeignore` contrôle ce que Claude Code peut voir/lire
- Tests dans `mobile/src/**/__tests__/` et `mobile/src/**/__mocks__/`
- Rapports dans `docs/bmad/`

## Étapes

### 1. Corriger `.claudeignore` (🔴 CRITIQUE)
Le fichier contient du wrapper PowerShell qui le rend malformé.
- **Supprimer** la ligne 1 : `@"`
- **Supprimer** la ligne 71 : `"@ | Set-Content .claudeignore`
- Le contenu entre ces deux lignes est le vrai contenu du fichier

### 2. Compléter `mobile/.easignore` (🟡)
Le fichier est trop minimal. Les fichiers suivants sont inutiles dans un build EAS et alourdissent l'upload :
Ajouter :
```
# Tests & mocks (inutiles dans le build)
**/__tests__/
**/__mocks__/
**/*.test.ts
**/*.test.tsx

# Coverage
coverage/

# Documentation
docs/
*.md

# Logs & temp
*.log
*.tmp

# Environment (géré par EAS secrets)
.env
.env.*

# IDE
.idea/
.vscode/

# BMAD
_bmad/
_bmad-output/

# Claude
.claude/
.claudeignore
```

### 3. Nettoyer `mobile/.gitignore` (🟡)
- Supprimer la ligne `.jks` (sans wildcard) car `*.jks` la couvre déjà
- Ajouter `coverage/` pour ignorer les rapports de couverture dans mobile/
- Ajouter `*.apk` et `*.aab` pour les builds locaux

### 4. Nettoyer `.gitignore` racine (🔵)
- Supprimer `.env.local` (ligne 41) car `.env*.local` (ligne 101) couvre déjà ce cas
- Ajouter `*.aab` (Android App Bundle produit par EAS)
- Ajouter `*.hprof` (heap dumps Android Studio)

### 5. Compléter `.vercelignore` (🔵)
Ajouter :
```
_bmad/
_bmad-output/
.claude/
.claudeignore
coverage/
*.log
```

## Contraintes
- Ne pas casser les builds EAS (`mobile/.easignore` ne doit PAS exclure des fichiers sources nécessaires au build)
- Ne pas exclure `mobile/src/` ou `mobile/assets/` de `.easignore`
- Ne pas modifier `web/.gitignore` (standard Next.js, déjà complet)
- Les `.idea/.gitignore` sont auto-générés par JetBrains — ne pas toucher

## Critères de validation
- Chaque fichier .ignore ne contient que des patterns valides (pas de syntaxe PowerShell ou autre)
- `git status` ne montre pas de nouveaux fichiers trackés qui devraient être ignorés
- Le build EAS n'est pas cassé (les fichiers source ne sont pas exclus)
