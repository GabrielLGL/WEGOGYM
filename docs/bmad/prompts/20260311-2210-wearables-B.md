# Rapport — Intégration wearables — Groupe B — 20260311-2210

## Objectif
Créer la couche service wearables : installer les packages natifs, définir une abstraction commune, implémenter Health Connect (Android) et HealthKit (iOS).

**Scope MVP :**
- **Health Connect** (Android — remplace Google Fit depuis Android 14)
- **HealthKit** (iOS — Apple Health)
- **Garmin** : hors scope MVP — requiert backend OAuth + webhooks serveur. À traiter séparément.

**Données synchronisées (MVP) :**
- Poids corporel → import auto dans `body_measurements`
- Durée/calories des séances de sport (lecture seule, comparaison)

## Fichiers concernés
- `mobile/package.json` — ajouter `react-native-health-connect` (Android) et `react-native-health` (iOS)
- `mobile/src/services/wearableService.ts` — **nouveau fichier** : abstraction commune + sélection auto par plateforme
- `mobile/src/services/healthConnectService.ts` — **nouveau fichier** : implémentation Android Health Connect
- `mobile/src/services/healthKitService.ts` — **nouveau fichier** : implémentation iOS HealthKit

## Contexte technique

### Packages à installer
```bash
cd mobile
npx expo install react-native-health-connect
npx expo install react-native-health
```

**Note** : `react-native-health-connect` requiert Android 14+ (API 34) ou l'app Health Connect installée sur Android < 14. Vérifier la compatibilité dans `app.json`.

### API Health Connect (Android)
```typescript
import {
  initialize,
  requestPermission,
  readRecords,
  HealthConnectPermission,
} from 'react-native-health-connect'

// Permissions à demander
const permissions: HealthConnectPermission[] = [
  { accessType: 'read', recordType: 'Weight' },
  { accessType: 'read', recordType: 'ExerciseSession' },
]

// Lecture des poids
const { records } = await readRecords('Weight', {
  timeRangeFilter: { operator: 'between', startTime, endTime },
})
// record.weight.inKilograms, record.time
```

### API HealthKit (iOS)
```typescript
import AppleHealthKit, { HealthValue } from 'react-native-health'

const permissions = {
  permissions: {
    read: [AppleHealthKit.Constants.Permissions.Weight, AppleHealthKit.Constants.Permissions.Workout],
  },
}

AppleHealthKit.initHealthKit(permissions, (error) => { ... })
AppleHealthKit.getLatestWeight({}, (err, result: HealthValue) => {
  // result.value (en livres par défaut, convertir en kg)
})
```

### Interface WearableService à implémenter
```typescript
interface WearableServiceInterface {
  // Disponibilité sur la plateforme
  isAvailable(): Promise<boolean>

  // Demander les permissions
  requestPermissions(): Promise<boolean>

  // Vérifier si les permissions sont accordées
  hasPermissions(): Promise<boolean>

  // Importer les pesées depuis la période donnée
  fetchWeightRecords(from: Date, to: Date): Promise<WeightRecord[]>

  // Importer les séances sport depuis la période donnée
  fetchWorkoutRecords(from: Date, to: Date): Promise<WorkoutRecord[]>
}

interface WeightRecord {
  weightKg: number
  timestamp: number  // unix ms
  source: 'health_connect' | 'healthkit'
}

interface WorkoutRecord {
  startTime: number
  endTime: number
  durationMinutes: number
  caloriesBurned?: number
  activityType: string
  source: 'health_connect' | 'healthkit'
}
```

### `wearableService.ts` — sélection auto par plateforme
```typescript
import { Platform } from 'react-native'
// Exporte une instance unique selon la plateforme
// Platform.OS === 'android' → HealthConnectService
// Platform.OS === 'ios' → HealthKitService
// Autre → null / stub vide
```

## Étapes

1. **Installer** les packages
   ```bash
   cd mobile
   npx expo install react-native-health-connect
   npx expo install react-native-health
   ```

2. **Créer** `mobile/src/services/healthConnectService.ts`
   - Implémente `WearableServiceInterface`
   - `isAvailable()` : vérifie si Health Connect est disponible
   - `requestPermissions()` : demande Weight + ExerciseSession
   - `fetchWeightRecords()` : lit les pesées sur la période
   - `fetchWorkoutRecords()` : lit les séances sport sur la période

3. **Créer** `mobile/src/services/healthKitService.ts`
   - Implémente `WearableServiceInterface`
   - Convertit les poids livres → kg
   - Gestion initHealthKit() + callbacks HealthKit

4. **Créer** `mobile/src/services/wearableService.ts`
   - Exporte `wearableService` (singleton sélectionné par Platform.OS)
   - Exporte les types `WeightRecord`, `WorkoutRecord`, `WearableServiceInterface`
   - Exporte la constante `WEARABLE_PROVIDER` ('health_connect' | 'healthkit' | null)

## Contraintes
- Pas de `any` TypeScript — typer toutes les réponses des APIs natives
- Pas de `console.log` sans `__DEV__`
- Gestion d'erreur robuste (permissions refusées, API non disponible, timeout)
- Conversions d'unités dans les services (livres → kg pour HealthKit)
- **Jamais** stocker les tokens/credentials en SQLite → `expo-secure-store`
- Garmin : NE PAS implémenter — hors scope, marquer un TODO commenté

## Critères de validation
- `npx tsc --noEmit` → zéro erreur
- `npm test` → zéro fail
- Les packages apparaissent dans `package.json`
- L'interface `WearableServiceInterface` est respectée par les deux implémentations
- `wearableService.ts` exporte correctement selon la plateforme
- Pas de `any` dans les signatures
