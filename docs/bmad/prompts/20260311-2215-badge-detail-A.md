# Rapport — Badge detail au clic — Groupe A — 20260311-2215

## Objectif
Quand l'utilisateur clique sur n'importe quel badge (verrouillé ou déverrouillé), afficher un BottomSheet de détail avec : titre, description motivationnelle, et la **condition pour l'obtenir** (ex : "Terminer 100 séances").

## Fichiers concernés
- `mobile/src/components/BadgeCard.tsx` (MODIFIER — ajouter prop `onPress`)
- `mobile/src/screens/BadgesScreen.tsx` (MODIFIER — gérer `onPress` + BottomSheet détail)
- `mobile/src/i18n/fr.ts` (MODIFIER — ajouter strings de condition)
- `mobile/src/i18n/en.ts` (MODIFIER — ajouter strings de condition)

## Contexte technique

### Patterns obligatoires
- **Modaux** : utiliser `<BottomSheet>` depuis `../components/BottomSheet` — JAMAIS `<Modal>` natif (crash Fabric)
- **Couleurs** : `const colors = useColors()` — jamais de couleurs hardcodées
- **i18n** : `const { t } = useLanguage()` — jamais de texte hardcodé
- **Haptics** : `const haptics = useHaptics()` — `haptics.onPress()` au clic
- **Modal state** : `useModalState()` pour open/close

### Comportement actuel (avant modif)
- `BadgeCard` a une prop `onLongPress` (uniquement sur badges déverrouillés) → ouvre ShareBottomSheet
- `BadgeCard` est `disabled` si pas de `onLongPress` → les badges verrouillés ne répondent à rien
- `BadgesScreen` gère le state `selectedBadge` + `shareSheet` pour le partage

### Comportement cible (après modif)
- **Clic simple** sur n'importe quel badge (verrouillé ou déverrouillé) → BottomSheet détail
- **Long press** sur badge déverrouillé → ShareBottomSheet (comportement existant, ne pas casser)
- BottomSheet détail affiche :
  - Icône du badge (grande taille)
  - Titre du badge (ex: "Centurion")
  - Description motivationnelle (ex: "100 séances d'entraînement. Respect.")
  - Condition d'obtention (ex: "Terminer 100 séances")
  - Si déverrouillé : badge vert "✓ Obtenu" ; si verrouillé : indication neutre

### Structure de `t.badges.list[badge.id]`
Chaque entrée a : `{ title: string, description: string }`. Pas de champ `condition` actuellement.

### `BadgeDefinition` (badgeConstants.ts)
```typescript
interface BadgeDefinition {
  id: string
  icon: string
  category: BadgeCategory  // 'sessions' | 'tonnage' | 'streak' | 'level' | 'pr' | 'session_volume' | 'exercises'
  threshold: number
}
```
Le threshold + category permettent de calculer la condition.

### i18n — pas d'interpolation native
Le système i18n custom ne supporte pas les fonctions ni `{n}`. Utiliser `.replace('{n}', ...)` en code :
```typescript
t.badges.conditions.sessions.replace('{n}', String(badge.threshold))
```

## Étapes

### 1. Ajouter les clés i18n dans fr.ts

Dans la section `badges`, ajouter un objet `conditions` :
```typescript
conditions: {
  sessions: 'Terminer {n} séances',
  tonnage: 'Soulever {n} 000 kg au total',
  streak: 'Maintenir un streak de {n} semaines',
  level: 'Atteindre le niveau {n}',
  pr: 'Battre {n} records personnels',
  session_volume: 'Faire {n} kg en une seule séance',
  exercises: 'Pratiquer {n} exercices différents',
  obtained: 'Obtenu ✓',
  locked: 'Non obtenu',
},
```

**Note tonnage** : le threshold est en kg brut (ex: 100_000 = 100 000 kg = 100 tonnes). Utiliser la valeur `threshold` directement et formater lisiblement. Le `{n}` sera remplacé par le threshold formaté. Pour `tonnage`, afficher en tonnes (threshold / 1000) ou en kg selon la taille :
- < 1000 : `${threshold} kg`
- ≥ 1000 : `${threshold / 1000} t`

Simplifier : utiliser `{n}` = valeur brute du threshold, formater dans le code.

En fait, pour éviter la complexité des unités, utiliser une clé plus simple :
```typescript
conditions: {
  sessions: '{n} séances complétées',
  tonnage_kg: '{n} kg soulevés au total',
  tonnage_t: '{n} tonnes soulevées au total',
  streak: '{n} semaines de streak',
  level: 'Niveau {n} atteint',
  pr: '{n} records personnels',
  session_volume: '{n} kg en une séance',
  exercises: '{n} exercices différents pratiqués',
  obtained: 'Obtenu',
  locked: 'À débloquer',
},
```

#### 4d. Créer la fonction helper de condition (inline dans le composant ou dans leaderboardHelpers)
```typescript
function getBadgeConditionText(badge: BadgeDefinition, t: typeof import('../i18n/fr').fr): string {
  const conditions = t.badges.conditions
  const n = badge.threshold

  if (badge.category === 'tonnage') {
    if (n >= 1000) {
      return conditions.tonnage_t.replace('{n}', String(n / 1000))
    }
    return conditions.tonnage_kg.replace('{n}', String(n))
  }

  const keyMap: Record<BadgeCategory, keyof typeof conditions> = {
    sessions: 'sessions',
    streak: 'streak',
    level: 'level',
    pr: 'pr',
    session_volume: 'session_volume',
    exercises: 'exercises',
    tonnage: 'tonnage_kg', // fallback (ne devrait pas arriver ici)
  }

  return conditions[keyMap[badge.category]].replace('{n}', String(n))
}
```

Cette fonction peut être définie en dehors du composant (module scope), en important les types nécessaires.

#### 4e. Ajouter le BottomSheet de détail dans le JSX

```tsx
{/* BottomSheet — Détail badge */}
<BottomSheet
  visible={detailSheet.isOpen}
  onClose={detailSheet.close}
  title={detailBadge
    ? (t.badges.list[detailBadge.id as keyof typeof t.badges.list]?.title ?? detailBadge.id)
    : ''}
>
  {detailBadge && (() => {
    const badgeI18n = t.badges.list[detailBadge.id as keyof typeof t.badges.list]
    const isUnlocked = unlockedIds.has(detailBadge.id)
    const conditionText = getBadgeConditionText(detailBadge, t)
    return (
      <View style={detailStyles.content}>
        {/* Icône grande */}
        <View style={[detailStyles.iconContainer, { backgroundColor: colors.card }]}>
          <Ionicons
            name={detailBadge.icon as ComponentProps<typeof Ionicons>['name']}
            size={48}
            color={isUnlocked ? colors.primary : colors.textSecondary}
          />
        </View>
        {/* Description motivationnelle */}
        {badgeI18n?.description && (
          <Text style={[detailStyles.description, { color: colors.textSecondary }]}>
            {badgeI18n.description}
          </Text>
        )}
        {/* Condition */}
        <View style={[detailStyles.conditionRow, { backgroundColor: colors.card }]}>
          <Text style={[detailStyles.conditionLabel, { color: colors.textSecondary }]}>
            🎯
          </Text>
          <Text style={[detailStyles.conditionText, { color: colors.text }]}>
            {conditionText}
          </Text>
        </View>
        {/* Statut */}
        <Text style={[
          detailStyles.status,
          { color: isUnlocked ? colors.primary : colors.textSecondary }
        ]}>
          {isUnlocked ? t.badges.conditions.obtained : t.badges.conditions.locked}
        </Text>
      </View>
    )
  })()}
</BottomSheet>
```

**Note** : éviter les IIFE si possible, extraire en composant inline ou utiliser une variable intermédiaire.

Meilleure approche sans IIFE :
```tsx
{detailBadge && (
  <BadgeDetailContent
    badge={detailBadge}
    isUnlocked={unlockedIds.has(detailBadge.id)}
    t={t}
    colors={colors}
  />
)}
```
Où `BadgeDetailContent` est un composant local défini dans le même fichier.
