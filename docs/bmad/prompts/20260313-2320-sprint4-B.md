# Rapport — Hall of Fame (Records all-time) — Groupe B — 20260313-2320

## Objectif
Créer un nouvel écran "Hall of Fame" qui affiche les meilleurs records personnels (PRs) de l'utilisateur pour chaque exercice, avec la date d'obtention et le 1RM estimé. Accessible depuis le bouton "Exercices & Records" de StatsScreen (ou via un nouveau bouton dédié dans la grille de Stats).

- `mobile/src/screens/StatsHallOfFameScreen.tsx` — NOUVEAU : écran principal

- `mobile/src/screens/StatsScreen.tsx` — MODIFIER : ajouter un bouton dans la grille

## Implémentation

### Structure de données
```ts
interface HallOfFameEntry {
  exerciseId: string
  exerciseName: string
  muscles: string[]
  bestWeight: number       // poids max tous PRs confondus
  bestReps: number         // reps associées au meilleur poids
  estimated1RM: number     // Epley : weight * (1 + reps/30)
  achievedAt: Date         // date du createdAt du set PR
}
```

### Logique de calcul (dans le composant avec `useMemo`)
1. Filtrer les sets avec `set.isPr === true`
2. Grouper par `set.exercise.id`
3. Pour chaque exercice, garder le set avec le meilleur `estimated1RM`
4. Trier par `estimated1RM` décroissant (top performers en premier)
5. Afficher max 50 entrées (les autres sont rarement consultées)

## Contraintes
- Pas de migration DB — utiliser uniquement les sets existants avec `isPr = true`
- Pas de tri par date uniquement — trier par `estimated1RM` décroissant
- `estimated1RM` = `Math.round(weight * (1 + reps / 30))`
- Pattern `withObservables` : écouter `sets` + `exercises` (sans `histories`)
- `useDeferredMount()` comme wrapper du composant observable
- Respecter `spacing.*`, `borderRadius.*`, `fontSize.*` du thème

## Critères de validation
- `npx tsc --noEmit` → zéro erreur TypeScript
- Le bouton apparaît dans StatsScreen grille
- La navigation vers StatsHallOfFame fonctionne
- Exercices sans PR (isPr jamais true) n'apparaissent pas
- Vide state affiché si aucun PR
