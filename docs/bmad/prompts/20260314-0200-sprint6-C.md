# Rapport — Comparateur de Périodes (#47) — Groupe C — 20260314-0200

## Implémentation

### Type de période prédéfinie
```ts
type ComparePeriod = 'this_month' | 'last_month' | 'last_3m' | 'last_6m' | 'this_year'
```

### Interface de stats d'une période
```ts
interface PeriodStats {
  label: string           // "Ce mois", "Mois dernier", etc.
  sessions: number
  volumeKg: number        // Σ(weight*reps) des sets dans la période
  prs: number             // count sets.isPr
  avgDurationMin: number  // moyenne des (endTime - startTime) / MINUTE_MS
  topMetric: 'sessions' | 'volume' | 'prs'  // quelle métrique est meilleure vs l'autre période
}
```

### Logique de calcul (dans useMemo)
```ts
function getPeriodRange(period: ComparePeriod): { start: number; end: number }
```
- `this_month` : du 1er du mois actuel à maintenant
- `last_month` : du 1er au dernier du mois précédent
- `last_3m` : depuis 90 jours
- `last_6m` : depuis 180 jours
- `this_year` : depuis le 1er janvier de l'année courante

Filtrer histories par `startTime >= start && startTime <= end`.
Filtrer sets par association avec ces histories (pré-construire `Set<historyId>` des histories retenues).

```

**Détails UI :**
- Chips de sélection période : style ChipSelector similaire (scrollable horizontal)
- Tableau comparatif : 4 lignes (séances, volume, PRs, durée), 3 colonnes (A, B, gagnant)
- Gagnant : icône `arrow-up-outline` en `colors.primary` si A > B, `colors.textSecondary` si égal
- Valeurs de la colonne gagnante en `colors.primary`, l'autre en `colors.textSecondary`
- Card résumé : phrase simple comptant les victoires A vs B

### withObservables pattern

const enhance = withObservables([], () => ({

      Q.or(Q.where('is_abandoned', null), Q.where('is_abandoned', false)),
    ))
  ).observe(),
  histories: database.get<History>('histories').query(
    Q.where('deleted_at', null),
    Q.or(Q.where('is_abandoned', null), Q.where('is_abandoned', false)),
  ).observe(),
}))
```
