# Rapport — i18n stats — Groupe B — 20260306-2300

## Objectif
Éliminer tous les fallbacks et labels FR hardcodés restants dans les utilitaires stats et écrans stats. Ajouter les clés de traduction manquantes dans fr.ts et en.ts.

## Fichiers concernés
- `mobile/src/model/utils/statsMuscle.ts`
- `mobile/src/model/utils/statsVolume.ts`
- `mobile/src/screens/StatsVolumeScreen.tsx`
- `mobile/src/screens/StatsMeasurementsScreen.tsx`
- `mobile/src/i18n/fr.ts`
- `mobile/src/i18n/en.ts`

## Contexte technique
- Les utilitaires stats sont des fonctions pures sans accès au contexte React → on passe les labels traduits en paramètre
- Plusieurs fonctions acceptent déjà des params optionnels (`dayLabels?`, `monthLabels?`) avec fallback FR — on les rend required ou on supprime le fallback
- Les écrans stats utilisent déjà `useLanguage()` → `{ t, language }` et peuvent passer les traductions aux utilitaires
