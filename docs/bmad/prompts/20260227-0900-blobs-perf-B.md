# Rapport — blobs-perf — Groupe B — 20260227-0900

## Objectif
Ajouter `overflow-hidden` aux containers parents de `BackgroundBlobs` pour que les blobs
(passés de `fixed` à `absolute` par le Groupe A) ne débordent pas hors du parent.

## Fichiers concernés
- `web/src/app/page.tsx`
- `web/src/app/privacy/page.tsx`

## Contexte technique
`BackgroundBlobs` est rendu dans un container `<div className="min-h-screen relative">` dans les deux fichiers.
Avec le passage de `fixed` à `absolute` (Groupe A), les blobs sont maintenant positionnés par rapport
à ce parent. Sans `overflow-hidden` sur le parent, les blobs (50vw × 50vw, partiellement en dehors)
pourraient créer une scrollbar horizontale ou déborder visuellement.

## Étapes

### Dans `web/src/app/page.tsx`
1. Repérer le container racine : `<div className="min-h-screen relative">`
2. Remplacer par : `<div className="min-h-screen relative overflow-hidden">`

### Dans `web/src/app/privacy/page.tsx`
1. Repérer le container racine : `<div className="min-h-screen relative">`
2. Remplacer par : `<div className="min-h-screen relative overflow-hidden">`

## Contraintes
- Ne modifier QUE la classe du container racine (le premier `<div>` de chaque page)
- Ne pas toucher aux autres éléments enfants (header, sections, z-index, etc.)
- Ne pas modifier `BackgroundBlobs.tsx` (géré par Groupe A)

## Critères de validation
- `npx tsc --noEmit` (depuis `web/`) → 0 erreur
- `npm test` (depuis `web/`) → 0 fail
- Vérifier visuellement qu'aucune scrollbar horizontale n'apparaît
- Les blobs restent confinés à la zone de page

## Dépendances
Aucune dépendance (Groupe A est indépendant)
