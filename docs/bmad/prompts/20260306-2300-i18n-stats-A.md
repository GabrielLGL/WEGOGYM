# Rapport — i18n stats — Groupe A — 20260306-2300

## Objectif
Remplacer les clés internes en français (`'1 mois'`, `'3 mois'`, `'Tout'`, `'Semaine'`) par des clés agnostiques de la langue (`'1m'`, `'3m'`, `'all'`, `'week'`). Supprimer `labelToPeriod()` devenue inutile.

## Fichiers concernés
- `mobile/src/model/utils/statsTypes.ts`
- `mobile/src/model/utils/statsDateUtils.ts`
- `mobile/src/screens/StatsVolumeScreen.tsx`

## Contexte technique
- L'app utilise un système i18n custom : `useLanguage()` → `{ t, language }` (cf `contexts/LanguageContext.tsx`)
- `PERIOD_LABELS` (statsTypes.ts ligne 5) est `['1 mois', '3 mois', 'Tout']` — utilisé comme clés internes ET comme props de `ChipSelector`
- `BAR_PERIOD_LABELS` (StatsVolumeScreen.tsx ligne 34) est `['Semaine', '1 mois', '3 mois', 'Tout']` — même problème
- `labelToPeriod()` (statsDateUtils.ts lignes 9-13) compare `label === '3 mois'` et `label === 'Tout'` pour retourner un `StatsPeriod`
- `StatsVolumeScreen` a déjà des maps `periodLabelMap` et `barPeriodLabelMap` qui traduisent les clés FR → `t.statsVolume.*` pour l'affichage. Le problème est que les clés elles-mêmes sont en FR.
- Les traductions existent déjà dans fr.ts/en.ts : `t.statsVolume.periodWeek`, `periodMonth`, `period3Months`, `periodAll`

## Étapes

### 1. `statsTypes.ts` — Refactorer PERIOD_LABELS
```typescript
// AVANT
export const PERIOD_LABELS = ['1 mois', '3 mois', 'Tout'] as const

// APRÈS
export const PERIOD_KEYS = ['1m', '3m', 'all'] as const
export type PeriodKey = typeof PERIOD_KEYS[number]
```

### 2. `statsDateUtils.ts` — Supprimer labelToPeriod()
La fonction `labelToPeriod()` convertit les labels FR en `StatsPeriod`. Avec des clés agnostiques, la conversion est directe (les clés SONT les `StatsPeriod`). Supprimer cette fonction.

Si elle est importée ailleurs, vérifier avec grep avant de supprimer.

### 3. `StatsVolumeScreen.tsx` — Refactorer les periods

**BAR_PERIOD_LABELS :**
```typescript
// AVANT
const BAR_PERIOD_LABELS = ['Semaine', '1 mois', '3 mois', 'Tout'] as const

// APRÈS
const BAR_PERIOD_KEYS = ['week', '1m', '3m', 'all'] as const
type BarPeriodKey = typeof BAR_PERIOD_KEYS[number]
```

**State :**
```typescript
// AVANT
const [periodLabel, setPeriodLabel] = useState<string>(PERIOD_LABELS[0])
const [barPeriodLabel, setBarPeriodLabel] = useState<string>(BAR_PERIOD_LABELS[0])

// APRÈS
const [periodKey, setPeriodKey] = useState<PeriodKey>(PERIOD_KEYS[0])
const [barPeriodKey, setBarPeriodKey] = useState<BarPeriodKey>(BAR_PERIOD_KEYS[0])
```

**Maps de traduction :**
Les maps `periodLabelMap` et `barPeriodLabelMap` changent de clés :
```typescript
const periodLabelMap: Record<PeriodKey, string> = {
  '1m': t.statsVolume.periodMonth,
  '3m': t.statsVolume.period3Months,
  'all': t.statsVolume.periodAll,
}
```

**ChipSelector :**
Le `ChipSelector` reçoit les clés agnostiques comme `items` et utilise un `labelExtractor` ou transforme les items en labels traduits avant de les passer. Vérifier la signature de ChipSelector pour la bonne approche.

**Conversion period :**
```typescript
// AVANT
const period = labelToPeriod(periodLabel)

// APRÈS — plus besoin de conversion, periodKey EST déjà un StatsPeriod
const period: StatsPeriod = periodKey
```
